# Root Python Script Archive

> 目的：收纳历史上散落在仓库根目录的 `_*.py` 临时脚本，减少根目录噪音，同时保留事故排查和版本发布历史。

---

## 分类

- `deploy/`
  - 历史版本发布、热修、性能验证相关脚本
- `verify/`
  - 部署后验证、链路验证、token/状态核验脚本
- `diag/`
  - 线上故障排查、批量任务排查、日志诊断
- `check/`
  - 本地检查、构建检查、CLI 检查
- `fix/`
  - 一次性数据修复、标签修复、状态修复
- `investigate/`
  - 查询、探索、轮询、临时调查脚本
- `other/`
  - 其余零散辅助脚本

## 使用建议

- 这里的大多数脚本都偏向“事故现场工具”，不应继续作为标准入口
- 若某类脚本被证明长期有价值，应重新抽成 `scripts/` 下的正式脚本，并补 README 或文档引用
- 新的一次性排障脚本建议直接放入对应分类目录，不要再落回仓库根目录
