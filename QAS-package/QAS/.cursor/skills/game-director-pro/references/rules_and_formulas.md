## 即梦 / Seedance 安全红线与替换词表

### 安全红线（所有提示词必须遵守）

| 禁止 | 替换为 |
|------|--------|
| 真人/本人/明星 | 3D 渲染角色、CGI、数字艺术形象 |
| 血腥/断肢/尸体 | 激烈动作、戏剧化冲突、神秘悬疑氛围 |
| 暴露/性暗示 | 优雅着装、得体服饰 |
| 迪士尼/原神/具体 IP 名 | 二次元赛博朋克风、中世纪幻想风、通用风格描述 |

### 提示词必包句式

- **中文**：`【原创数字艺术作品】` + [画面描述] + `，适合全年龄段观看，无版权争议，无敏感元素`
- **英文**（API 用）：`Original digital artwork, [description], family-friendly, no copyright issues, no sensitive content.`

## Viral Hook 公式库（前 3 秒必用其一）

### 好奇型 (Curiosity)
- 关于这个隐藏 Boss，99% 的玩家都错过了……
- 这把绝版武器的获取方式，居然这么简单？
- 新地图里有个彩蛋，触发方式绝了

### 故事型 (Story)
- 当硬核玩家误入休闲模式……
- 这曾经是全服公认的最废职业，直到……
- 从开荒到毕业，我只用了三天

### 价值/干货型 (Value)
- 拒绝刮痧！当前版本最强开荒阵容：
- 别再盲目抽卡了，这 3 个角色才是真神
- 三个技能秒 Boss，配置直接抄

### 反直觉型 (Contrarian)
- 都在吹新地图，只有我觉得这关的设计才是神作？
- 都说这角色弱，我偏要拿他上分

## 宣传向视频公式（新角色 / 新地图）

### 公式 4：新角色 — CG 史诗宣发型

*适用：角色 PV、角色预告。参考：浪人预告结构。*

| 时长占比 | 内容要求 |
|----------|----------|
| **0–40%**（电影级铺垫） | 高质量 CG/剧情过场。画面偏暗或神秘感。角色配音宿命感台词。**无花字，纯享视听**。 |
| **40%–80%**（实机爽点） | 音乐爆发/变奏，带 UI 的实机战斗混剪。多技能释放（大范围 AoE、高伤数字）。**极简花字：技能名**。 |
| **80%–90%**（收招定格） | 切回 CG，角色收刀或背影。一句压迫感/挑衅台词收束。 |
| **90%–100%**（CTA） | 游戏 Logo + 双端下载按钮，沉重打击音效。 |

### 公式 5：新地图 — 沉浸探索揭秘型

*适用：新地图展示、探索揭秘。参考：冰雪大世界结构。*

| 时长占比 | 内容要求 |
|----------|----------|
| **0–25%**（奇观 Hook） | 大远景/宏伟建筑/奇观地貌。空灵、大气环境音与史诗 BGM。 |
| **25%–65%**（多职业跑图） | 多职业背影向地标行进，人物与场景比例差展示纵深。**职业名称烫金花字**。 |
| **65%–85%**（地下城/危机） | 转入室内/地牢/祭坛，光线变暗，紧张压抑，暗示高收益与高风险。 |
| **85%–100%**（CTA） | 游戏 Logo + 下载引导，音乐余音渐弱。 |

## 视频脚本结构模板（分镜阶段必参照）

做**新英雄**或**新地图**宣传片分镜时，按下方模板划分阶段并填写「画面 / 声音 / 核心目的」，再拆成镜号填入生产包表格。

### 新英雄模板（对应公式 4）

| 阶段 | 画面内容 (Visual) | 声音设计 (Audio) | 核心目的 |
|------|-------------------|------------------|----------|
| **0–5 秒** | 吸睛宏大场景 / 标志性建筑物 / 角色背影特写。冷色调。 | 低沉的 BGM 起，环境音（如风声、乌鸦叫）。主角低语。 | 留住用户的滑动动作。 |
| **5–20 秒** | 武器特写 / 战火纷飞的闪回 / 角色眼神特写。冷暖色对比。 | 压迫感增强的 BGM，重低音鼓点。主角念白交代身份或宿命。 | 建立情感连接与期待感。 |
| **21–50 秒** | 切入实机画面（去掉冗余 UI，只留血条或伤害数字）。快速剪辑 3–5 个华丽连招和终极技能。 | 激昂的战斗 BGM，拳拳到肉的打击音效，技能释放的爆破音。 | 展示核心玩法，释放爽点。 |
| **50–60 秒** | 角色处决动作或帅气收招，画面定格。切入游戏 Logo。 | BGM 骤停或推向最高潮，干净利落的 UI 音效，强有力的结束语。 | 引导下载。 |

### 新地图模板（对应公式 5）

| 阶段 | 画面与内容要点 | 核心目的 |
|------|----------------|----------|
| **前 10 秒 · 静谧与宏大** | 大远景或缓慢推镜，展示最具标志性、最高规格的美术奇观（宏伟城堡、诡异巨像等）。**不出现战斗**，只用环境音（风声、水滴声）。 | 建立奇观 Hook，吸引停留。 |
| **10–30 秒 · 地貌多样性快剪** | 配合音乐鼓点，快速切换不同生态地貌（雪山 → 沼泽 → 熔岩 → 森林），展示世界广阔度。 | 传达地图体量与多样性。 |
| **30–45 秒 · 微观细节与动态环境** | 展示地图内动态元素：随风摇摆的植被、墙壁上忽明忽暗的火把、机关陷阱运作，体现「鲜活感」。 | 强化沉浸与探索欲。 |
| **45–60 秒 · 实机跑图/潜行** | 视角拉回玩家，展示在精美地图中探索、潜行或互动的实机画面；结尾用「推开大门走向未知新地图」的镜头收束。 | 实机代入感 + 引导下载/预约。 |

## 宣传向 vs 病毒向 剪辑侧重

| 维度 | 宣传向 | 病毒向 |
|------|--------|--------|
| 节奏 | 稳，突出角色/场景/技能 | 快切、强卡点、前 3s 抓人 |
| 音乐 | 史诗/氛围/品牌感 | 热门 BGM、重低音、梗曲 |
| 花字/文案 | 角色名、技能名、卖点 | Hook 句、梗、反问、悬念 |
| 目标 | 转化、预约、认知 | 完播、点赞、转发、讨论 |

## 角色一致性与 I2V 首帧

### 为什么多镜同角色会不一致？

- **文生视频（T2V）**：每个镜头独立生成，模型不记忆上一镜，同一「浪人武士」在不同镜头会被画成不同长相、服装、武器细节。  
- **图生视频（I2V）**：以**同一张角色图**作为首帧，模型在首帧基础上做动作/运镜，**角色外观会明显一致**。

### 为什么不能直接用「本地路径」或「Google Drive 分享页」？

- **本地路径 / UNC**（如 `C:\...\浪人.png` 或 `\\10.21.100.89\...\浪人.png`）：  
  视频生成 API 跑在**云端服务器**上，无法访问你电脑上的 C 盘或公司内网共享盘，所以传本地路径没有用，必须传一个**服务器能通过互联网访问的 URL**。

- **Google Drive**：  
  分享链接通常是「预览页」或带跳转/登录的页面，很多 API 需要的是「一次 GET 请求就能拿到图片原始字节」的**直链**。Drive 若用「知道链接的人可查看」并配合直链格式（如 `https://drive.google.com/uc?export=download&id=FILE_ID`），在部分情况下可用，但容易因重定向、限流或返回 HTML 而失败，**不如对象存储（TOS/OSS/S3）或图床稳定**。可以尝试，若 API 报错再改用图床/OSS。

- **结论**：只要能让 API 服务器**用 HTTP GET 直接拿到图片内容**的 URL 即可；可以是公网图床、对象存储、或可用的网盘直链。

### 推荐流程（同角色多镜）

1. **用户已提供本地角色参考图**（如 浪人立绘.png、素材库路径）时，**必须优先用该图保证服化道与画风一致**：  
   - **首帧/尾帧产出**：不要用文生图 API 为「角色镜」重新画一遍（会导致每镜角色、画风不一致）；改为将**参考图复制/缩放**为各角色镜的 `shot_XX_first_frame.png` 与 `shot_XX_last_frame.png`。  
   - 在批量生成时使用：`scripts/generate_frame_images.py --input <json> --output-dir <dir> --reference-image <参考图路径>`，并在 JSON 或参数中指定 `use_reference_for_shots`（角色镜镜号，如 01、02、03、06），脚本会自动用参考图填充这些镜的首帧与尾帧。  
   - **画风一致**：整片角色造型、服装、武器、色调均来自同一张素材，避免文生图每张一个风格。  
2. **素材库有角色图**（如 `Gold And Glory_Characters/浪人.png`）时，在分镜表中标明「可作 I2V 首帧」；若为本地/UNC 路径，同上，用 `--reference-image` 复制为角色镜首尾帧。  
3. **需要公网 URL 做 I2V 时**：将参考图上传至能提供**直链**的服务（火山 TOS、阿里 OSS、七牛、ImgBB 等），得到 `https://...`，对应镜头使用 **image-to-video**，传入 `image_url` = 该链接。  
4. **同一角色多镜**（如镜 1、2、3、6）**共用同一张首帧/参考图**，可最大程度保持造型与画风一致。

### 无法提供 I2V 时的兜底：角色设定卡

当暂时无法提供首帧 URL、只能 T2V 时，在**每个镜头的提示词**中**固定加入同一段角色外观描述**，减少各镜之间的随机差异（仍无法保证与素材库立绘一致）。  

**示例 — 浪人角色设定卡（按参考图提炼，可粘贴进各镜提示词）**：

```
同一角色设定：日本浪人武士，宽檐草帽（笠）遮住上半脸，下巴与嘴可见；内层浅色/白和服宽袖，肩胸深蓝/黑外层，蓝灰腰带、左侧垂下，腰间褐色绳状/草编质感；双手深色指套（露指），深色宽袴，传统草履；右手持打刀，刀鞘深色、刀尖向下。3D 写实渲染，质感清晰。所有镜头保持该造型一致。
```

**首帧与尾帧衔接（必守）**：同一镜的首帧 = 该镜的**起幅**（动作/场景的开始一帧），尾帧 = **同一场景、同一角色**的**落幅**（结束一帧）。写 prompt 时首帧写「谁在哪、起始状态」，尾帧写「同一人、同一处、结束状态」，保证可接成一段镜头，禁止首尾各说各的、换人换景。

使用时在画面描述前插入角色设定，例如：  
`【原创数字艺术作品】[上述角色设定]。浪人背对镜头站在雨夜废墟与城楼前，刀鞘轮廓清晰……，适合全年龄段观看，无版权争议，无敏感元素。`

## BGM 与音频提示词产出（成片合成时必出）

在 STAGE 2.5 / 3.5 须产出**整片 BGM + 整片音频提示词**。结构见 SKILL.md 内「BGM 与音频提示词产出规范」；简要模板如下：

- **整片 BGM 提示词**：风格 + 器乐 + 情绪/节奏（前中后段 + BPM）+ 时长 + 卡点与分镜对齐 + 英文短句（供 Suno/Udio 等）。
- **上周热门**：WebSearch 查 `TikTok trending music this week` / `Reels viral sound` 等，列出 3–5 首并注明平台与适用段落。
- **自创 AI 音乐**：同上整片提示词，可按国内/国外 AI 作曲工具再拆成中文或英文即用版。
- **整片音效提示词**：按镜号或时间轴列出每条音效的「时间位置 + 类型 + 描述词 + 可选英文」，供 AI 音效生成或素材库查找。
- **旁白/对白提示词**（若有）：按时间列出「时间 + 文案 + 语气 + 可选 TTS 描述词」，供配音或 TTS 使用。

为其他角色做多镜时，可从素材库参考图或设定稿中提炼：**发型、服装主色与款式、武器/道具、整体风格**，写成一段固定「角色设定卡」，在所有该角色的镜头提示词中统一加入。
