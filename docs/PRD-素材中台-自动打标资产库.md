# PRD：GOBS 素材中台（自动打标资产库）

## 1. 产品目标

让游戏项目组从"找素材、传素材、记素材"中解放出来。
用户在 GOBS 的理想体验是：

> 上传一次素材包 → 系统自动整理并打标 → 在视频生成/剪辑中秒选可用素材 → 一键出多版本内容

### 成功标准（体验指标）
- 新用户首次导入后，3 分钟内能找到可用素材并开始生成
- 用户在生成前找素材步骤从"多次切换网盘/聊天/本地"降到"单页面完成"
- 80% 常用素材可通过标签检索命中（无需手动翻找）

---

## 2. 目标用户与场景

### 用户角色
1. **运营同学**：每天出社媒/买量素材，最怕找不到图和视频
2. **制片同学**：需要角色一致、场景一致，最怕素材混乱
3. **剪辑同学**：有大量录屏，最怕手动挑片

### 核心场景
- **项目启动**：一次性导入历史素材包
- **日常生产**：每天新增素材自动归档并打标
- **内容产出**：生成和剪辑直接调用"项目资产库"

---

## 3. 产品定位

不是"文件管理器"，而是"可直接用于创作的素材中台"。

---

## 4. 核心用户旅程（体验优先）

### Journey A：首次建库（Onboarding）
1. 进入"项目资产库"
2. 拖入文件夹/zip（支持大量文件）
3. 系统自动分析并展示进度（可后台跑）
4. 用户进入"待确认标签"页，快速批量确认
5. 完成后立即看到"可用于生成""可用于剪辑"的推荐素材

**关键体验点**
- 上传中可离开页面，不丢任务
- 失败项可单独重试
- 导入完成后给"下一步行动"按钮：去生成 / 去剪辑

### Journey B：生成时找素材
1. 在 Studio/高级制片点击"从资产库选择"
2. 先按"用途"筛（买量/社媒/版本宣发）
3. 再按"角色/场景/风格/镜头"筛
4. 一键加入 prompt 参考（图/视频）

**关键体验点**
- 默认给"推荐素材"，不是让用户先搜索
- 所见即所得（缩略图 + 标签 + 适用平台）

### Journey C：剪辑时找片段
1. 在 Editor 打开"项目资产库"侧边栏
2. 输入"吃鸡高光/团战/爆点"关键词
3. 返回视频候选 + 高光时间点
4. 拖到时间轴直接落片

**关键体验点**
- 按"可剪片段"粒度返回，不只是整条视频
- 有"质量提示"（模糊、水印、竖版适配）

---

## 5. 功能设计

### 5.1 导入与解析
- 支持：文件夹、zip、多文件拖拽、云盘导入（后续）
- 自动去重：hash + 文件名相似度
- 自动抽取元数据：分辨率、时长、帧率、方向、音轨

### 5.2 自动打标（双层）
- **规则标签（稳定）**：横竖版、时长区间、清晰度、水印风险
- **AI 标签（语义）**：角色、场景、动作、情绪、用途建议

每个 AI 标签带 confidence，低置信进入"待确认"。

### 5.3 人工确认台（必须有）
- "待确认"队列
- 批量改标签
- 一键应用到相似素材
- 全部操作支持撤销

### 5.4 素材浏览与检索
- **维度筛选**：角色 / 场景 / 类型 / 用途 / 比例 / 时长
- **语义搜索**："热血战斗开场""搞笑瞬间"
- **预设视图**：
  - 可用于买量（9:16 + 高冲突画面）
  - 可用于社媒（日常剧情向）
  - 可用于版本更新（功能展示清晰）

### 5.5 与生成/剪辑打通
- **生成侧**：选择素材自动写入提示词结构
- **剪辑侧**：素材拖入时间轴 + 候选高光点快速插入

---

## 6. 信息架构（IA）

- 一级导航新增：**项目资产库**
- 子页面：
  1. 资产总览
  2. 导入任务
  3. 待确认标签
  4. 素材检索
  5. 使用记录（哪些素材被哪些视频使用）

---

## 7. 关键交互原则（易用核心）

1. 默认推荐优于空白搜索
2. 批量操作优于逐条编辑
3. 状态清晰优于静默后台（上传中/分析中/待确认/可用）
4. 创作闭环优于孤立管理（能直接去生成/剪辑）
5. 低门槛优先（第一次导入无需理解复杂标签体系）

---

## 8. 非功能要求（体验相关）
- 导入任务可断点续传
- 分析任务可暂停/继续
- 标签变更有操作日志
- 多用户隔离（按 username + project 维度）
