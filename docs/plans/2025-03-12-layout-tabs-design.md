**日期**: 2025-03-12  
**状态**: 待评审  
**设计风格**: 沿用当前 Deepix 深色主题（紫主色）

## 一、目标结构

### 1.1 最外围：三个 Tab

| Tab | 路由 | 默认 | 说明 |
|-----|------|------|------|
| **视频生成** | `/` 或 `/generate` | ✅ 默认 | 主流程：输入 prompt → 一键优化 → 一键匹配素材 → 开始生成 |
| **素材管理** | `/materials` | 否 | 设置 Drive 素材库、查看已选素材 |
| **视频分发** | `/distribute` | 否 | 成片列表、发布到社媒（Phase 2 壳子） |

**首次进入**：用户进入「视频生成」tab。

### 1.2 Tab 形式

**方案 A（推荐）**：顶部水平 Tab 栏，紧贴页面顶部，占一层。

```
┌─────────────────────────────────────────────────────────────┐
│ [视频生成]  [素材管理]  [视频分发]     QAS 视频工坊           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  主内容区（根据当前 Tab 切换）                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**方案 B**：沿用当前左侧边栏，Tab 置于侧栏顶部。

建议采用 **方案 A**，与截图和「最外围三个 tab」的表述一致。

## 二、视频生成 Tab 内容（主流程）

### 2.1 布局（自上而下）

```
┌─────────────────────────────────────────────────────────────┐
│  开启你的 视频生成 即刻造梦!                                 │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [+ 参考内容]  在此输入 prompt，支持 @图片1 @视频1...  │   │
│  │  (上传区)    或粘贴模糊创意，点击下方「一键Prompt」   │   │
│  │              自动优化为可用的视频描述                 │   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  [配置行] 模型: Seedance 2.0  |  比例: 16:9  |  时长: 5s    │
├─────────────────────────────────────────────────────────────┤
│  [一键Prompt]  [一键匹配素材]  [开始生成]                   │
├─────────────────────────────────────────────────────────────┤
│  下方区域：                                                  │
│  - 一键Prompt 成功后：展示优化后的 prompt、关键词标签        │
│  - 一键匹配素材 成功后：展示素材网格、可勾选排序             │
│  - 开始生成 点击后：分镜步骤（可折叠）+ 视频生成进度         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 三个按钮逻辑

| 按钮 | 行为 | 前置条件 | 状态 |
|------|------|----------|------|
| **一键 Prompt** | 调用 `POST /api/prompt/polish`，优化输入区文字，展示优化结果+关键词 | 输入区有文字 | 无输入时置灰 |
| **一键匹配素材** | 基于关键词在 Drive 中搜索，展示素材网格供选择 | ① 素材库已设置（见 2.3）<br>② 有关键词（可来自一键 Prompt 或手动输入） | 未设置素材库时：提示「请先在素材管理设置 Drive 文件夹」并引导跳转 |
| **开始生成** | 触发：分镜生成 → 视频生成（或两步合并） | ① 一键 Prompt 已执行成功<br>② 一键匹配素材已执行且用户已选至少 1 个素材 | **不满足时置灰** |

### 2.3 素材库首次设置

**方案 A**：点击「一键匹配素材」时若未设置，弹 Modal 或内联卡片：「请先在素材管理设置 Drive 文件夹」，提供「去设置」跳转到 素材管理 tab。

**方案 B**：素材管理 tab 作为唯一入口，用户必须先访问一次。视频生成 tab 的「一键匹配素材」在未设置时置灰，tooltip：「请先在素材管理设置 Drive 文件夹」。

**推荐：方案 A** — 主动引导，减少迷失。

## 五、设计优化建议（对照截图与 UX）

### 5.1 推荐优化

| 项目 | 建议 | 理由 |
|------|------|------|
| **按钮层次** | 「开始生成」为主按钮（更突出），另外两个为次级 | 符合主操作优先级 |
| **配置行** | 可折叠，默认展开；移动端可收起到「高级」 | 减少首屏信息量，保持简洁 |
| **一键匹配素材 未设置时** | 不置灰，可点击；点击后弹出引导「去素材管理设置」 | 避免用户不知为何置灰 |
| **进度反馈** | 一键 Prompt / 一键匹配素材 执行中显示 loading，禁用按钮 | 防重复提交 |
| **空状态** | 输入区为空时，placeholder 明确说明流程：输入创意 → 一键优化 → 匹配素材 → 生成 | 降低学习成本 |

### 5.2 保持现状

- Deepix 深色主题、紫色主色
- 输入区支持多行、支持 @图片N 占位说明
- 素材选择：勾选 + 顺序，映射 @图片1 @图片2

### 5.3 可选增强（非必须）

- 参考内容上传区（+ 参考内容）：当前以文字为主，上传可作 Phase 2
- 配置行：模型、比例、时长 — 可先写死或简单下拉，后续接 API

## 六、路由与状态

### 6.1 路由

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | TabGenerate | 视频生成（默认） |
| `/materials` | TabMaterials | 素材管理 |
| `/distribute` | TabDistribute | 视频分发 |
| `/result` | Result | 成片预览（可从生成流程跳转） |

### 6.2 状态（CreateFlowContext 扩展）

```ts
// 新增
hasPolishedPrompt: boolean    // 一键 Prompt 已成功执行
hasMatchedMaterials: boolean // 一键匹配素材已执行且已选至少 1 个
```

用于控制「开始生成」的 enabled 状态。

## 七、实现顺序建议

1. **布局**：新增顶部 Tab 栏，替换当前侧边栏；路由改为 `/`, `/materials`, `/distribute`
2. **视频生成 tab**：重组为「输入区 + 配置行 + 三按钮 + 下方结果区」；接入已有 polish、search、Drive 逻辑
3. **素材管理 tab**：迁移 Step 1（Drive 设置）+ 已选素材展示
4. **视频分发 tab**：占位壳子
5. **按钮状态**：实现 hasPolishedPrompt、hasMatchedMaterials，控制「开始生成」置灰
