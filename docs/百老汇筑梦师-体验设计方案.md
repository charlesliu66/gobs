# 百老汇筑梦师 — 沉浸式体验设计方案

> Broadway Dreammaker：把 AI 视频创作平台重构为一座活的剧院。
> 用户不是在操作软件，而是在一座剧院里创造一场演出。

---

## 一、设计哲学

### 1.1 核心隐喻

**平台 = 一座百老汇剧院**

用户是导演兼编剧，AI 是幕后剧组（灯光师、道具师、摄影师、剪辑师）。每一次创作都是"一场演出的诞生"——从灵感到剧本，从排练到首演，从首演到巡回。

### 1.2 三条第一原则

| 原则 | 含义 | 反面 |
|------|------|------|
| **叙事驱动** | 用户的创作流程是一条故事线，不是功能菜单 | 不做"侧栏→页面→表单"的工具箱 |
| **作品是主角** | 视频/分镜/角色永远是屏幕上最大的视觉元素，UI 退到幕后 | 不让控件面板占据主视觉区 |
| **空间即状态** | 用户通过"身处哪个房间"感知自己在做什么，而非看导航高亮 | 不依赖侧栏/面包屑来定位 |

### 1.3 心理模型对比

```
当前模型（工具箱）:
  用户 → 打开功能 → 填表/操作 → 看结果 → 关闭

目标模型（剧院）:
  用户 → 走进一个空间 → 被环境引导 → 演出在眼前成型 → 走向下一个空间
```

## 二、空间架构

### 2.1 四幕结构

整个平台由四个核心空间组成，对应创作的四个阶段。用户在空间之间的移动遵循线性叙事，但可以自由回溯。

```
  ┌─────────────────────────────────────────────────────────────────┐
  │                        🎭 剧院                                  │
  │                                                                 │
  │   ┌────────────┐                                                │
  │   │  大厅       │  ← 常驻入口：所有演出的海报墙                   │
  │   │  The Lobby  │     创建新演出 / 回顾旧演出                     │
  │   └─────┬──────┘                                                │
  │         │                                                       │
  │         ▼ 选择/创建一场演出                                      │
  │                                                                 │
  │   ┌─────────────────────────────────────────────────────────┐   │
  │   │                 演出工作流（线性四幕）                     │   │
  │   │                                                         │   │
  │   │   第一幕            第二幕           第三幕      谢幕     │   │
  │   │  ┌──────────┐   ┌──────────┐   ┌──────────┐  ┌──────┐  │   │
  │   │  │ 编剧室    │ → │ 排练厅   │ → │ 首演夜   │→ │巡演厅│  │   │
  │   │  │ Writer's  │   │Rehearsal │   │ Premiere │  │On    │  │   │

  │   │                                                         │   │
  │   └─────────────────────────────────────────────────────────┘   │
  │                                                                 │
  └─────────────────────────────────────────────────────────────────┘
```

### 2.2 空间关系规则

- **大厅是枢纽**：任何时候都可以"回到大厅"
- **四幕是线性的**：首次创作必须从第一幕开始，但回溯自由
- **幕间有门**：从一幕走到下一幕有明确的"过门"交互（不是自动跳转）
- **每幕有自己的光线和氛围**：编剧室昏暗温馨，排练厅明亮专业，首演夜绚烂，巡演厅辽阔

## 三、逐空间详细设计

### 3.1 大厅（The Lobby）

**替代**：当前首页 + 项目列表

**概念**：用户登录后进入的第一个空间。这不是一个 dashboard，而是一座剧院的大厅——暗色调、穹顶隐约可见、两侧是发光的演出海报。

#### 海报墙交互细节

**海报状态映射**：

| 项目状态 | 海报视觉 | 说明 |
|---------|---------|------|
| 刚创建，无内容 | 铅笔草图轮廓，微微发光 | "想法刚诞生" |
| 有脚本，未生成 | 水彩渲染，静态画面 | "剧本已写好，等待排练" |
| 有生成视频 | 活的——循环播放精彩片段 | "演出就绪" |
| 已发布 | 金色边框，底部有巡演标记 | "正在巡演中" |
| 归档 | 褪色老照片效果 | "经典剧目" |

**交互**：

- **Hover**：海报缓缓向前浮出（translateZ + scale 1.05），周围环境变暗（vignette），海报发出微弱的聚光灯光晕
- **点击**：聚光灯全亮 → 海报放大至全屏 → 灯光暗转 → 进入该演出对应的当前幕
- **新建（中央光环）**：一束垂直聚光灯照射地面的圆形光斑。hover 时光斑变亮并向上延伸。点击后，一张空白海报从光中缓缓升起 → 弹出"给你的演出起个名字"

**底部快速输入**：

- 一个低调的输入框，placeholder: "今晚上演什么？"
- 用户直接输入一个创意 → 自动创建新演出并带着这段话进入编剧室
- 输入框下方有 4-5 个灵感标签（如"一场雨夜追逐"、"猫咪宇航员的故事"），点击直接填入

#### 导航

大厅**没有传统导航栏/侧栏**。空间本身就是导航：
- 海报 → 进入演出
- 右上角微妙的个人头像 → 账号/设置
- 左上角剧院 logo → 回到大厅（从任何地方）
- 底部极低调的数据摘要

#### 概念墙交互

**元素类型**：

| 元素 | 创建方式 | 视觉 | 作用 |
|------|---------|------|------|
| 故事便签 | 输入文字 | 黄色便签纸，手写体 | 故事梗概/场景描述 |
| 角色卡 | 上传图片 or AI 生成 | 竖向卡片，头像+名字+描述 | 角色定义 |
| 风格参考 | 上传图片 or 选预设 | 图片卡，底部标签 | 视觉风格锚定 |
| 音乐情绪 | 选预设 or 上传 | 波形卡片，可播放 | BGM 方向 |
| AI 建议 | AI 自动生成 | 半透明蓝色便签 | 可采纳/忽略 |

**核心交互**：

1. **自由布局**：便签可拖动，可堆叠，可分组。不强制结构——让用户以自己的方式整理创意
2. **AI 共创**：用户每添加一个元素，AI 会在 2-3 秒后在附近"贴出"一个相关建议（淡入出现，不打断）
3. **角色生成**：在角色卡上写描述 → 点击"画个像" → AI 生成角色立绘，直接替换占位图
4. **连线**：可以在便签之间画线，表示"这个角色出现在这个场景"——AI 会用这些连线理解故事结构

#### "门"机制

编剧室的右侧（或底部进度条的"排练"节点）有一扇门。

- **门关着（灰暗）**：还没有足够的素材进入排练。条件：至少有 1 个故事便签 + 1 个角色或风格参考
- **门微亮（琥珀色光从缝隙透出）**：素材充足，可以进入排练
- **门打开（金色光涌出）**：用户 hover 到门上时，门缓缓推开，透出排练厅的灯光
- **走进去**：点击门 → 屏幕暗转（幕间） → 编剧室的元素在黑暗中飞散重组 → 亮起时已经变成了排练厅的分镜

### 3.3 排练厅（Rehearsal Hall）— 第二幕

**替代**：当前的分镜表 + 视频生成 + 剪辑工作台（三合一）

**概念**：一个开阔的排练空间。上半屏是舞台（预览区），下半屏是控制台（时间轴/参数）。用户在这里经历三个连续阶段：走位 → 彩排 → 精修，不切换页面。

#### 三个子阶段

**阶段 A — 走位（Blocking）**

对应当前：分镜表编辑

| 维度 | 设计 |
|------|------|
| 舞台区 | 当前选中镜头的全画幅预览（静态分镜图 or 已生成的视频） |
| 控制台 | 横向排列的分镜卡片条，每张卡片显示：缩略图 + 镜号 + 简述 |
| 编辑 | 点击卡片 → 舞台聚焦到这一镜 → 右侧滑出参数面板（从侧幕滑入） |
| 参数面板 | 标题: "灯光师的建议" — 运镜、光线、节奏、时长，每个参数有 AI 推荐标记 |
| 排序 | 拖拽卡片调整顺序，卡片之间有微妙的连接线（像舞台走位的标记线） |
| AI | 舞台左侧偶尔弹出"舞台监督"的建议气泡："第3镜和第4镜的景别重复了" |

**阶段 B — 彩排（Rehearsal）**

对应当前：视频生成（批量）

| 维度 | 设计 |
|------|------|
| 触发 | 用户点击"开始彩排"（一个大幕拉起的按钮动效） |
| 舞台区 | 自动播放模式——从第一镜开始，每生成完一个镜头就自动播放 |
| 控制台 | 每张分镜卡片显示生成状态：⏳等待 → 🎬拍摄中 → ✅完成 → ❌NG |
| 进度感 | 不是一个进度条，而是卡片依次"亮起来"——像灯泡逐个点亮 |
| NG 重拍 | 点击卡片上的 ❌ → "重新排练这场" → 该镜头重新生成，其他不受影响 |
| 等待体验 | 舞台区在等待时显示"幕间"画面：暗色背景+旋转的聚光灯+当前进度文案 |

**等待文案序列**（不是固定的，根据进度滚动）：

```
"灯光师在调整光位……"
"演员正在走位……"
"道具组在布景……"
"摄影师在找角度……"
"导演喊了一声 Action——"
```

**阶段 C — 精修（Fine Cut）**

对应当前：剪辑工作台

| 维度 | 设计 |
|------|------|
| 触发 | 所有镜头生成完毕后，控制台区域平滑过渡为时间轴 |
| 舞台区 | 保持——播放区域，与时间轴同步 |
| 控制台 | 标准时间轴编辑器（当前剪辑工作台的核心 UI），但视觉语言融入剧院主题 |
| 过渡 | 从彩排模式的卡片序列 → 动画变形为时间轴上的片段（卡片拉长、对齐、拼接） |
| 工具面板 | 从侧幕（左侧或右侧）滑入：转场库、BGM 库、文字样式 |
| AI | "剪辑师"角色："第2镜和第3镜之间加个叠化转场会更流畅" |

#### 舞台区的视觉设计

```
┌──────────────────────────────────────────────┐
│           ┌───── 顶部幕布 ─────┐              │
│           │  ▼▼▼▼▼▼▼▼▼▼▼▼▼▼  │              │
│  ┌─────┐  │                    │  ┌─────┐    │
│  │侧幕 │  │                    │  │侧幕 │    │
│  │条   │  │   视频预览画面      │  │条   │    │
│  │     │  │   (主角)           │  │     │    │
│  │参数 │  │                    │  │AI   │    │
│  │面板 │  │                    │  │建议 │    │
│  │(滑出)│  │                    │  │(弹出)│    │
│  └─────┘  │                    │  └─────┘    │
│           └────────────────────┘              │
│              ▲▲▲ 舞台前沿 ▲▲▲                 │

- 视频预览区域有微妙的"舞台框"装饰——顶部有帷幕暗纹，两侧有柱子暗影
- 不能太重——这是暗示，不是主题公园。80% 是功能，20% 是氛围
- 参数面板从左侧"侧幕"位置滑出，AI 建议从右侧"侧幕"探出

---

### 3.4 首演夜（Premiere Night）— 第三幕

**替代**：当前的导出页面 + 连续播放审片

**概念**：排练完成，精修就绪。这是"大幕拉开"的时刻——用户的作品首次以完整形态呈现。

#### 视觉与交互

```
┌────────────────────────────────────────────────────────┐

#### 核心体验

1. **进入首演夜时**：屏幕全暗 1s → 大幕拉开动画（CSS clip-path 从中央向两侧展开）→ 视频自动开始播放
2. **播放中**：全屏沉浸，无 UI 干扰。底部有极淡的进度条。按 Esc 可退出
3. **播放结束**：灯光渐亮（背景从纯黑渐变为暖金），出现行动选择
4. **情感设计**：这个"大幕拉开→首演→掌声"的仪式感，是让用户对自己的作品产生骄傲感的关键时刻

## 四、贯穿全局的系统设计

### 4.1 幕间过渡系统

空间切换不是"页面跳转"，而是**幕间过渡**：

```
当前空间灯光渐暗（0.3s）
  → 短暂黑屏 + 过渡文案（0.5s）
    → "第二幕 · 排练"
  → 新空间灯光渐亮（0.4s）
```

过渡文案示例：
- 大厅 → 编剧室: "灵感降临……"
- 编剧室 → 排练厅: "剧本已就位，开始排练。"
- 排练厅 → 首演夜: "一切就绪。大幕即将拉开——"
- 首演夜 → 巡演厅: "掌声未落，世界在等。"

### 4.2 导航系统（替代侧栏）

**没有传统侧栏**。导航嵌入在空间体验中：

**底部幕间条（Curtain Bar）**：

```
┌──────────────────────────────────────────────────────────┐
│  ◉ 编剧 ────────── ◉ 排练 ────────── ○ 首演 ──── ○ 巡演  │
│   ██████            ██░░░░                                │
└──────────────────────────────────────────────────────────┘
```

- 始终停留在屏幕最底部，高度极小（40px）
- 当前幕用发光圆点标记
- 已完成的幕用实线连接
- 未解锁的幕用虚线连接
- Hover 时展开为略高的面板，显示每幕的简要信息

**全局返回**：
- 左上角 logo（或剧院名）→ 点击回到大厅
- 键盘快捷键 `Esc` → 回到大厅

### 4.3 AI 角色系统

AI 不是一个统一的"助手"，而是根据当前空间扮演不同角色：

| 空间 | AI 角色 | 人格 | 出现方式 |
|------|---------|------|---------|
| 编剧室 | 联合编剧 | 发散、有创意、偶尔大胆 | 黑板上自动出现的蓝色便签 |
| 排练厅（走位） | 舞台监督 | 严谨、注重连贯性 | 侧幕探出的对话气泡 |
| 排练厅（彩排） | 制片人 | 关注进度和质量 | 底部状态栏的文字提示 |
| 排练厅（精修） | 剪辑师 | 技术导向、有品味 | 时间轴上方的建议条 |
| 巡演厅 | 经纪人 | 了解平台规则、营销思维 | 每个平台节点旁的建议 |

AI 建议的通用交互：
- 非打断式——总是从边缘淡入，不阻塞当前操作
- 可采纳/忽略——明确的两个按钮
- 可关闭——"安静模式"暂时屏蔽 AI 建议

### 4.4 色彩系统

```
                暖色 ←──────────────────→ 冷色
                  │                        │
  大厅          琥珀金 #f5c542              │
  编剧室        暖黄  #d4a847              │
  排练厅                    靛蓝紫 #8b7dff  │
  首演夜        大幕红 #c4354a   →   金色庆典
  巡演厅                           冷青  #22d3ee
```

| Token | 暗色值 | 用途 |
|-------|--------|------|
| `--theater-void` | `#0a080f` | 最深背景（灯灭状态） |
| `--theater-dark` | `#14111e` | 标准背景（座席区） |
| `--theater-surface` | `#1e1a2e` | 卡片/面板（舞台面） |
| `--theater-curtain` | `#c4354a` | 大幕红——重要 CTA |
| `--theater-gold` | `#f5c542` | 金色点缀——聚光灯/高亮/accent |
| `--theater-spotlight` | `#8b7dff` | 主色——当前焦点/primary |
| `--theater-text` | `#f0ede4` | 暖白文字（聚光灯下的纸张色） |
| `--theater-muted` | `#9b95b0` | 次要文字（薰衣草灰） |
| `--theater-dim` | `#5d5875` | 最暗文字（后排座位的暗影） |

亮色模式：保持剧院隐喻但转为日场排练——
- 背景：暖米白 `#faf8f4`（像排练厅的木地板在自然光下）
- 表面：纯白
- 文字：深紫棕 `#2d2540`

### 4.5 字体系统

```
标题字体:   DM Serif Display（现代衬线，有戏剧感但不古板）
            或 Playfair Display（经典百老汇海报衬线）
正文字体:   Plus Jakarta Sans（保留，清晰通用）
标签/数据:  Space Grotesk（保留，工具感）
特殊:       手写效果用于编剧室便签（CSS font-style: italic + 轻微 rotate）
```

标题层级示例：

| 场景 | 字体 | 大小 | 效果 |
|------|------|------|------|
| "今晚上演什么？" | DM Serif Display | 48px | 金色，居中，微妙 text-shadow |
| "第二幕 · 排练" | DM Serif Display | 20px | 暖白，letter-spacing 0.15em |
| 分镜参数标签 | Space Grotesk | 11px | uppercase, 薰衣草灰 |
| 便签文字 | Plus Jakarta Sans | 14px | italic, 微妙 rotate(-1deg) |

### 4.6 动效原则

| 动效类型 | 时长 | 缓动 | 用途 |
|---------|------|------|------|
| 幕间过渡 | 1.2s | ease-in-out | 空间切换 |
| 聚光灯 focus | 0.4s | ease-out | hover 高亮 |
| 侧幕滑入 | 0.35s | cubic-bezier(0.16, 1, 0.3, 1) | 面板出现 |
| 海报浮出 | 0.5s | cubic-bezier(0.34, 1.56, 0.64, 1) | 卡片 hover（带微弹） |
| 灯光渐变 | 0.8s | ease-in | 环境光变化 |
| 大幕拉开 | 2.0s | cubic-bezier(0.4, 0, 0.2, 1) | 首演夜开幕 |
| 卡片亮灯 | 0.3s | ease-out | 生成完成时逐个亮起 |

## 七、设计风险与对策

| 风险 | 严重程度 | 对策 |
|------|---------|------|
| 装饰过重，影响功能效率 | 高 | 排练厅（工作核心）装饰最少，保持工具效率；氛围主要在大厅和过渡 |
| 学习成本增加——新用户不知道怎么操作 | 中 | 首次进入有"导览"：一个虚拟的"剧院引座员"带你走一遍 |
| 幕间过渡太慢，影响高频用户 | 中 | 提供"快速通道"——键盘快捷键可以跳过过渡动效 |
| 移动端难以还原空间感 | 高 | 移动端简化为线性卡片流，保留色彩和文案主题，放弃空间过渡 |
| "百老汇"隐喻对中国用户有距离感 | 低 | 中文界面用"筑梦剧院"而非"百老汇"；核心是"剧院"这个通用隐喻 |
| 性能——过渡动效 + 视频预览同时存在 | 中 | 海报墙视频预览用 IntersectionObserver 懒加载，仅可见区域播放 |

## 八、与现有 DESIGN.md 的关系

当前已有的 `DESIGN.md`（Framer + RunwayML 融合版）定义了底层设计 token 和组件规范。百老汇筑梦师方案是在其上的**主题层覆盖**：

- **保留**：组件结构（按钮、输入框、卡片的 HTML 结构不变）
- **替换**：色彩 token（冷蓝→暖紫金）、字体（加入衬线标题）、动效（加入幕间系统）
- **新增**：空间过渡系统、海报墙组件、概念墙组件、底部幕间条
- **移除**：传统侧栏、功能菜单式导航

具体实施时，可以通过 CSS 变量层覆盖实现主题切换，保持代码层面的灵活性。
