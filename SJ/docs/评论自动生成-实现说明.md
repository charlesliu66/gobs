# 评论自动生成 · 实现说明

对应需求：`docs/评论自动生成-需求确认书.md`。

## 功能概览

- **规则库**：`GET/POST/PATCH/DELETE /api/comment-rules`；纯文本多条规则（新建时用 `rulesText`，服务端用 `splitRulesFromInput` 拆条），可附 `.md`/`.py` 文件；`PATCH` 可更新 `name` / `rules` / `files`（含重命名）。
- **规则库上传 `.py` / `.md`**：持久化在 `CommentRuleLibrary.files`；生成时由 `flattenRulesForGeneration` 将**文件全文**并入提示词（与纯文本 `rules` 一起发给模型），**不依赖**磁盘上的 `commander.py`。`.py` 在提示中会包在代码块内并说明「按注释/常量中的策略语义遵守」。
- **可选：全局 `commander.py`（Safety Filter）**：若希望在所有规则库之外再加一层，可在磁盘放置 `commander.py`（或 `COMMENT_COMMANDER_PY_PATH`）；`mergeGlobalCommanderIntoRules` 会将其**置于规则列表最前**，与规则库上传文件**叠加**生效。
- **生成**：`POST /api/comment-rules/generate`，入参为 `ruleLibraryId`、`prompt`、`language` 及可选任务上下文；使用 `OPENAI_API_KEY` 调用 Chat Completions。
- **批量评论页**：`control-panel.tsx` 首页仅保留「选用规则库」下拉与提示词；**新建 / 列表 / 重命名 / 查看与删除** 均在 **「管理规则库」** 对话框内；解析粘贴后若已选规则库且填写提示词，则逐条调用 generate；表格中每条评论旁 **刷新** 为重生成该条。

## 持久化

- 与任务状态相同：优先 **Upstash / Vercel KV**（`KV_REST_*` 或 `UPSTASH_*`）；否则本地 **`web/.data/comment-rule-libraries.json`**（已 gitignore 的 `.data/` 目录）。
- **Vercel 无持久磁盘**：未绑 KV 时，规则库仅在单次实例内存或不可用，**生产请务必配置 KV**。

## 环境变量

| 变量 | 说明 |
|------|------|
| `GEMINI_API_KEY` 或 `GOOGLE_API_KEY` | **优先**：Google Gemini（`AIzaSy...`），用于规则库生成与 Sparkles 备选 |
| `GEMINI_MODEL` | 可选，默认 `gemini-2.0-flash` |
| `OPENAI_API_KEY` | 未配置 Gemini 时使用 |
| `OPENAI_API_BASE` / `OPENAI_MODEL` | 可选 |
| `KV_REST_API_URL` + `KV_REST_API_TOKEN` | 推荐（规则库持久化） |
