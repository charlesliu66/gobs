## 一、核心 Loop 总览

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                        Quality at Scale 核心流水线                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

  [1] Prompt 输入
        │
        ▼
  ┌─────────────────────────────────────────────────────────────────────────────┐
  │ [2] 内容检索 + 脚本生成                                                        │
  │     基于 prompt 检索本地文件 → 生成结构化视频脚本（分镜、时间码、Seedance 格式）   │
  └─────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
  ┌─────────────────────────────────────────────────────────────────────────────┐
  │ [3] 视频制作（Seedance）                                                       │
  │     脚本 + 素材 → 自动生成视频                                                  │
  └─────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
  ┌─────────────────────────────────────────────────────────────────────────────┐
  │ [4] 分发（Distribute）                                                         │
  │     成片 → 批量发布到多平台（需解决：自动创号 / 多账号管理）                       │
  └─────────────────────────────────────────────────────────────────────────────┘
```

## 二、各阶段拆解

### 阶段 1：Prompt → 检索 → 结构化脚本

| 环节 | 输入 | 输出 | 关键问题 |
|------|------|------|----------|
| **1.1 检索** | Prompt（主题/关键词） | 相关本地文件列表 | 检索什么？文档？素材？两者都要？ |
| **1.2 脚本生成** | Prompt + 检索结果 | 结构化视频脚本 | 脚本格式？与现有 video-director 如何衔接？ |

**已确认（2025-03-10）**：
- **当前**：本地文件 = **素材（图片）**，用于 Seedance 的 @图片N 引用
- **示例**：prompt「浪人打怪物的10秒视频」→ 检索「浪人」「怪物」相关图片 → 映射 @图片1、@图片2 → 生成脚本
- **后期扩展**：可加入产品世界观文档，约束产出的视频在设定框架内（角色设定、场景规则等）

---

### 阶段 2：视频制作（Seedance）

与现有 **video-pipeline** 基本一致：
- 输入：结构化 prompt + 素材路径
- 执行：`node run.js --prompt "..." --materials "..." --duration N`
- 输出：成片到 `Ai Videos/`

**可复用**：video-director（分镜规范）+ video-pipeline（自动化执行）

---

### 阶段 3：分发（Distribute）

| 环节 | 输入 | 输出 | 关键问题 |
|------|------|------|----------|
| **3.1 平台选择** | 成片 + 配置 | 目标平台列表 | ✅ 已定：**INS、TikTok、YouTube、Facebook** |
| **3.2 账号管理** | - | 可用的发布账号 | 使用 **GeeLark 云手机**：多设备独立指纹，模拟真实手机 |
| **3.3 发布** | 成片 + 文案 + 云手机 | 各平台已发布链接 | GeeLark API + 自动化模板（TikTok/Instagram/YouTube/Facebook） |

**已确认（2025-03-10）**：
- **目标平台**：Instagram、TikTok、YouTube、Facebook
- **技术方案**：GeeLark 云手机（虚拟手机，多设备独立指纹，支持批量发布）

## 三、关键决策点

### 决策 1：检索的「本地文件」类型 ✅ 已定

| 选项 | 说明 | 状态 |
|------|------|------|
| **当前：仅素材** | 检索图片/视频，用于 Seedance @图片N。例：浪人、怪物 → 浪人.png、怪物.png | ✅ 采用 |
| **后期：+ 世界观文档** | 加入产品设定文档，约束脚本在角色/场景/规则框架内 | 📌 预留 |

---

### 决策 2：检索方式

| 选项 | 说明 | 复杂度 |
|------|------|--------|
| **A. 关键词/文件名匹配** | 从 prompt 提取关键词，在文件名中匹配 | 低，与 video-pipeline 一致 |
| **B. 向量检索（RAG）** | 文档做 embedding，语义检索 | 中，需向量库 |
| **C. 混合** | 素材用文件名，文档用向量 | 高 |

---

### 决策 3：分发阶段的账号策略 ✅ 已定

| 方案 | 说明 | 状态 |
|------|------|------|
| **GeeLark 云手机** | 每台云手机独立设备指纹，可预先登录各平台账号，通过 API 批量发布 | ✅ 采用 |
| **API 集成** | 成片 → 调用 GeeLark API → 批量上传到云手机 → 执行自动化发布到 INS/TikTok/YouTube/Facebook | ✅ 采用 |

## 四、推荐架构（初版）

基于你已有的 video-director + video-pipeline，建议分步实现：

### 第一步：强化「Prompt → 脚本」链路

```
Prompt（例：浪人打怪物的10秒视频）
  → 解析意图（video-director Step 1）：时长、主体、动作、风格
  → 按 prompt 关键词检索素材库（浪人、怪物 → 匹配文件名）
  → 素材映射 @图片1、@图片2...（复用 video-pipeline 模式 A）
  → 生成分镜脚本（video-director Step 2-4，含 Seedance 规则审核）
  → 输出 Seedance 可用 prompt

[后期] 若配置世界观文档：检索文档 → 抽取设定约束 → 注入脚本生成（角色一致性、场景规则等）
```

### 第二步：视频制作

- 直接复用 video-pipeline 的 `run.js` 自动化

### 第三步：分发（GeeLark 集成）

```
成片（Ai Videos/*.mp4）
  → 上传到 GeeLark 素材库（File Management API）
  → 调用 Automation API：批量分发到云手机
  → 执行发布任务：TikTok / Instagram Reels / YouTube Shorts / Facebook
  → 可配置：发布时间、文案、标签
```

**GeeLark 能力**：
- 云手机：多台独立设备，每台可登录不同平台账号
- API：云手机管理、文件上传、自动化任务、Webhook 回调
- 支持：TikTok、Instagram Reels、YouTube Shorts、Facebook 等 10+ 平台

## 五、GeeLark 集成说明

### 能否「操作」GeeLark？

| 方式 | 说明 |
|------|------|
| **直接操作 GUI** | ❌ 不能。Cursor Agent 无法点击/打开 GeeLark 桌面客户端 |
| **通过 API 集成** | ✅ 可以。编写脚本调用 GeeLark API，实现程序化：创建云手机、上传视频、触发发布任务 |

### 集成方式

- 使用 GeeLark API（https://open.geelark.com/api）：POST 请求，JSON 格式
- 接口：文件管理、云手机管理、自动化任务、Webhook
- 可编写 Python/Node.js 脚本，在 Quality at Scale 流水线末尾调用，实现「成片 → 自动分发」
