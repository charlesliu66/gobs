# Planner Spec - AOV 自然语言混剪适配

## 1) 目标

让“自然语言混剪”可稳定适配 `Arena of Valor (AOV)`：  
用户输入中文指令（如“给我做一个澜 12 秒打野节奏混剪，强调龙团和收割”），系统能按 AOV 规则完成镜头检索、规则判定、时间线编排、字幕与包装输出。

## 2) 背景与外部事实

- AOV 官方有明确模式体系（5v5、排位等）与玩法入口，说明“模式差异”是规则配置核心。来源：<https://www.arenaofvalor.com/m/game_modes_list.shtml>
- 社区/攻略中对 AOV 常见宏观规则一致：三路、野区、Abyssal Dragon、Dark Slayer、团战资源优先级。来源：  
  - <https://samurai-gamers.com/arena-of-valor/lane-guide/>  
  - <https://gamingonphone.com/guides/arena-of-valor-dark-slayer-lane-guide/>
- 短视频混剪主流做法强调：节拍对齐、击杀/团战高光提取、竖屏快节奏包装（更贴近 Shorts/TikTok 生态）。来源：<https://capscutfree.com/capcut-for-gaming-montages/>

## 3) In Scope / Out of Scope

### In Scope

1. AOV 项目级规则包（模式、术语、事件权重、禁用词、字幕词典）。
2. AOV 专用内容识别层（事件检测 + 术语识别 + HUD/OCR 识别）。
3. 自然语言意图解析到“混剪执行计划”（时长/风格/节奏/重点事件）。
4. 时间线编排器支持 AOV 模板（开场 Hook -> 节奏推进 -> 高光爆点 -> 收束 CTA）。
5. 可观测与回放：为何选中这些片段、为何判定为“龙团/收割/反打”。

### Out of Scope（本期不做）

1. 所有 MOBA 一次性全适配（只先做 AOV）。
2. 高级语义检索模型训练（先规则+轻量模型混合）。
3. 全自动语音解说生成（仅留扩展点）。

## 4) 模块拆分（你还要开发的功能）

### M1. 项目规则中心（Project Ruleset）

- `game_profile`：`aov` 项目元数据（地区、语言、分辨率、HUD 版本）。
- `mode_rules`：5v5/排位/娱乐模式的判定差异（事件阈值、镜头偏好）。
- `taxonomy`：英雄、分路、野怪、资源点、状态词（开龙、控线、反野、收割、反打）。
- `policy`：内容合规词典（平台敏感词替换、字幕安全模板）。

### M2. AOV 内容识别引擎

- 视觉检测：
  - HUD 关键区 OCR（击杀播报、连杀、目标击杀提示、时间轴）
  - 小地图/资源点状态弱检测（可先半自动标注）
- 事件抽取：
  - 击杀、连杀、团战爆发、开龙/拿龙、推塔、逆风翻盘片段
- 角色与语义：
  - 英雄名别名映射（中英文、常见简称）
  - “收割/反打/绕后”等术语映射到可计算事件组合

### M3. 自然语言 -> 混剪 DSL 规划器

- 输入示例：“做一个 15 秒 AOV 打野节奏片，前 3 秒必须爆点，重点龙团和三连击杀。”
- 输出 DSL：
  - `duration=15s`
  - `structure=[hook, buildup, climax, outro]`
  - `must_events=[dragon_fight, triple_kill]`
  - `style=[speed_ramp, beat_sync, impact_caption]`
- 失败兜底：当检索不到目标事件时，返回可解释降级方案（如改“团战高光优先”）。

### M4. AOV 时间线编排器（Montage Composer）

- 支持规则模板：
  - `竞技高光`（击杀驱动）
  - `运营逆转`（目标资源+推进驱动）
  - `英雄秀`（单英雄镜头连续性）
- 关键能力：
  - 节拍对齐与切点优化
  - 速度曲线（慢放爆点 + 常速回切）
  - 字幕自动生成（事件词 + 英雄词）

### M5. 评估与可观测（必须）

- `trace`：每个片段的入选理由与规则命中记录。
- `quality score`：节奏密度、事件密度、重复镜头率、可读性。
- A/B 支持：不同模板输出对比。

## 5) 技术方案（建议）

1. **规则优先 + 轻量模型增强**  
   先上线可控规则链（规则中心 + 事件提取 + DSL 编排），再在关键点用模型提分（事件分类、镜头美学排序）。
2. **项目隔离配置**  
   所有判定策略挂在 `project_rules/aov/*.yaml`，避免逻辑硬编码。
3. **可回放管线**  
   每次混剪都保存中间产物（候选片段、过滤原因、排序分数、最终时间线）。

## 6) 风险清单

- R1: AOV 版本/HUD 更新导致 OCR 失效。  
  缓解：HUD 模板版本化 + 快速回滚。
- R2: 术语识别歧义（如“开龙”具体指哪种资源）。  
  缓解：语义词典带上下文别名和优先级。
- R3: 素材噪声高（压缩、遮挡、非标准画面）。  
  缓解：置信度阈值 + 降级策略 + 人工确认入口。
- R4: 平台合规风险（字幕与描述词）。  
  缓解：输出前敏感词扫描与替换。

## 7) 验收标准（AC）

- AC-01: 可创建 AOV 项目规则包并版本化发布。
- AC-02: 可从原视频中识别至少 6 类 AOV 事件（击杀/连杀/团战/开龙/推塔/高光时刻）。
- AC-03: 自然语言输入可稳定生成 DSL 计划，失败有可解释降级。
- AC-04: 生成 9:16 与 16:9 两种混剪成片，节拍和字幕可配置。
- AC-05: 输出 trace 面板，可追踪每个片段的选中依据。
- AC-06: 对同一素材支持至少 3 种 AOV 模板（竞技高光/运营逆转/英雄秀）。
- AC-07: 对无目标事件素材，系统不崩溃，返回“可发布备选方案”。

## 8) 测试矩阵

- T-01 (AC-01): 规则包创建/更新/回滚。
- T-02 (AC-02): 事件识别准确率抽检与混淆矩阵。
- T-03 (AC-03): 20 条中文自然语言指令解析一致性。
- T-04 (AC-04): 两种画幅输出与字幕布局检查。
- T-05 (AC-05): trace 完整性与可解释性验证。
- T-06 (AC-06): 模板切换回归测试。
- T-07 (AC-07): 空结果与低置信度降级路径压力测试。
