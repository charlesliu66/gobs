# 优化功能需求说明（确认后开发）

请先过目以下需求描述，确认无误后再进行开发。

---

## 一、批量粘贴增强

**现状**：批量粘贴区仅支持「视频链接 + 评论」（Tab/逗号分隔），解析后填入表格。

**需求**：支持在粘贴内容中一并包含 **时区、发布时间、发布日期**，减少逐行填写成本。

**格式约定（待你确认）**：
- 每行一条任务，列顺序建议：**视频链接 | 评论 | 发布日期 | 发布时间 | 时区**
- 分隔符：Tab（便于从 Excel 复制）或逗号；若用逗号，评论内容中若有逗号需约定转义或改用 Tab。
- 示例（Tab 分隔）：
  ```
  链接	评论内容	2025-03-25	14:30	UTC+8
  ```
- 若某列省略：时区默认用「默认时区」、日期/时间可留空由用户在表格中补填。

**请确认**：列顺序、分隔符（仅 Tab / 仅逗号 / 两种都支持）、日期时间格式（如 `YYYY-MM-DD`、`HH:MM`）是否按上述执行，或有其他要求？

---

## 二、控制台 · 设备台页签（云手机管理）

**需求**：在控制台新增「**设备台**」页签（或独立菜单），将 GeeLark API 中的 **云手机管理** 能力集成到同一页面，便于集中管理设备。

**需集成的能力（参考 en-doc 云手机管理接口）**：
- 云手机列表：分页/条件查询、展示状态（开机/关机/启动中等）
- 开机 / 关机（单台或批量）
- 创建云手机（含一键新机等）
- 修改云手机（名称、备注等）
- 删除云手机
- 转移 / 分组（若有对应 API）
- 其他文档中与「Cloud Phone Management」相关的读写在设备台内完成

**请确认**：是否按上述范围集成即可，是否有「必须优先做」或「暂不做的」子项？

## 三、任务日志

**需求**：
- **记录任务状态**：对「批量发评论」等任务做状态记录与展示（如：等待中、执行中、成功、失败、已取消）。
- **任务集合与子任务**：一个「批量评论任务」视为一个任务集合，其下包含多条子任务（每条对应一个链接）；日志中能查看该集合整体进度及各子任务状态。
- **可终止任务**：支持对未完成或执行中的任务（或整个集合）进行「终止」操作，并调用 GeeLark 取消任务接口。
- **评论任务链接可点击**：列表中每条评论任务对应的「视频链接」为可点击链接，点击可新开标签页打开该 TikTok 链接。

**请确认**：任务数据来源是否以 GeeLark 任务查询接口（如 `task/query`、`task/historyRecords`）为主？是否需要「仅记录前端发起的任务」还是也要展示「在 GeeLark 控制台或其他端创建的任务」？

## 四、设置 · 代理管理

**需求**：在「**设置**」页签（或设置模块）中，对代理进行 **增删改查**，与 GeeLark 代理管理 API 对接。

**功能点**：
- **查询**：分页/列表展示代理，支持按 ID 等条件筛选（与 API 一致）。
- **新增**：填写代理信息并调用 GeeLark 添加代理接口。
- **修改**：编辑已有代理并调用更新接口。
- **删除**：删除指定代理并调用删除接口。
- 若 API 支持「代理检测」，可在列表中或详情中提供「检测」按钮并展示结果。

**请确认**：代理字段（类型、地址、端口、用户名、密码等）是否完全按 GeeLark 文档的请求参数来？是否有额外展示或校验需求？

## 五、新增「应用」页签（GeeLark 自动化能力）

**需求**：新增「**应用**」页签，集中展示 **GeeLark 已集成好的自动化功能**（如：TikTok 发视频、养号、多平台分发、Instagram/Facebook/YouTube 等自动化任务类型），便于用户按类型选择并创建/管理自动化任务。

**可包含内容（参考 en-doc）**：
- 按平台或任务类型分类展示（如：TikTok、Instagram、Facebook、YouTube、多平台分发等）。
- 每类下列出对应 GeeLark API 能力（如：发视频、登录、养号、发评论等），可点击进入「创建任务」或说明页。
- 若某功能已在本项目中有单独页面（如批量评论），可在此提供入口跳转。

**请确认**：本页签以「导航/入口汇总」为主，还是需要部分能力在本站内直接完成创建与提交（若为后者，请说明优先做哪几类）？
