# Planner Spec - unified project lifecycle

> Run: `2026-04-23-unified-project-lifecycle`
> Gate 1 artifact

---

## Goal

统一优化剪辑器与高级制片的项目生命周期，让两端都先停留在临时草稿态，只有在形成有效内容并完成强制命名后才转正为正式项目，同时为历史未命名项目提供批量治理入口。

## Scope

### In

- 剪辑器首次保存前的强制命名转正逻辑
- 高级制片首次服务端保存前的强制命名转正逻辑
- 两端未命名项目批量治理入口
- 后端首次创建项目时的未命名防回归保护

### Out

- 自动后台迁移历史未命名项目
- 收藏、归档、置顶等长期项目管理能力
- 大规模重做项目列表视觉样式

## Acceptance Criteria

| ID | 场景 | 预期 |
|---|---|---|
| AC-1 | 用户进入剪辑器后不做正式命名 | 不会产生新的未命名正式项目 |
| AC-2 | 用户进入高级制片后开始编辑但未命名 | 不会产生新的未命名正式项目 |
| AC-3 | 两端草稿第一次达到有效内容阈值 | 必须先命名，命名成功后才转正入库 |
| AC-4 | 用户打开项目列表治理未命名项目 | 能批量删除空项目并批量为有内容项目生成建议名 |
| AC-5 | 未来其他入口绕过前端校验 | 后端仍能拦截新的空名称项目创建 |

## Risks

- 高级制片当前存在本地草稿和服务端项目双轨持久化，若拦截点选错，可能导致本地草稿也无法续写
- 剪辑器当前自动保存与 URL project 参数联动较深，若改动不慎可能影响最近项目自动打开
- 批量治理如果误判“空项目”，会造成误删风险，因此需要保守判定

## Approach

1. 先抽公共 lifecycle helper，统一“未命名识别 / 有效内容判定 / 建议命名”
2. 再分别接入剪辑器和高级制片首存拦截
3. 最后补后端 guard rail 和批量治理入口
