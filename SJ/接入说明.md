# GeeLark 控制台工程 — 接入说明

本工程为 **Next.js 前端 + GeeLark API** 的 TikTok 批量评论 / 云手机管理 / 任务日志 控制台。同事可将其接入到自己的 Cursor 工程中按需使用。

> 解压后请阅读本文件并按「二、接入方式」操作。

---

## 一、包内结构说明

```
SJ/
├── web/                    # Next.js 应用（主工程）
│   ├── app/                # 页面与 API 路由
│   ├── components/         # React 组件
│   ├── lib/                # GeeLark 调用、解析等
│   ├── .env.local.example  # 环境变量示例（必读）
│   └── package.json
├── docs/                   # 需求/说明文档（可选参考）
├── .env.example            # 根目录环境变量示例（Python 用，可选）
└── 接入说明.md             # 本文件
```

---

## 二、接入方式（二选一）

### 方式 A：整包放入 Cursor 工程目录

1. 将解压后的 **`web` 文件夹** 复制到你的 Cursor 工程下（例如 `你的项目/geelark-console/`）。
2. 在终端进入该目录：
   ```bash
   cd 你的项目/geelark-console
   ```
3. 复制环境变量并填写 Token：
   ```bash
   cp .env.local.example .env.local
   ```
   编辑 `.env.local`，至少填写：
   ```env
   GEELARK_BEARER_TOKEN=你的Bearer_Token
   GEELARK_BASE_URL=https://openapi.geelark.com/open/v1
   ```
4. 安装依赖并启动：
   ```bash
   npm install
   npm run dev
   ```
5. 浏览器访问：`http://localhost:3000`。

### 方式 B：只复用前端代码与 API

1. 从本包中按需复制到你的工程：
   - `web/app/` → 你的 `app/`（或对应 Next 路由目录）
   - `web/components/` → 你的 `components/`
   - `web/lib/geelark.ts`、`web/lib/parse-batch-paste.ts`、`web/lib/time.ts` 等 → 你的 `lib/` 或工具目录
2. 在你的工程中增加与 `web/.env.local.example` 一致的环境变量（如 `GEELARK_BEARER_TOKEN`）。
3. 安装本工程用到的依赖（见 `web/package.json` 的 `dependencies`），然后 `npm run dev` 运行你的工程。

---

## 三、环境变量（必配）

在 **`web/.env.local`** 中配置（可参考 `web/.env.local.example`）：

| 变量 | 必填 | 说明 |
|------|------|------|
| `GEELARK_BEARER_TOKEN` | 是 | GeeLark 开放平台 Bearer Token |
| `GEELARK_BASE_URL` | 否 | 默认 `https://openapi.geelark.com/open/v1` |
| `OPENAI_API_KEY` | 否 | 用于 AI 生成评论，不填则用内置备选 |

---

## 四、主要功能

- **批量评论定时发布**：多视频链接、多评论，批量粘贴（Tab 分隔），可选亚洲版 API。
- **任务命名**：提交后可输入本次任务名称，在任务日志中展示，便于回溯。
- **设备台**：云手机列表、启停、编辑备注与代理。
- **任务日志**：本站批量任务（含单条终止）、GeeLark 近期任务、任务报告（详情 / 云手机 / 计划与耗时 / 失败码与原因 / 日志 / 截图）。
- **设置**：代理的增删改查（按 GeeLark 文档参数）。

---

## 五、注意事项

- 不要将 **`.env`、`.env.local`** 或任何含 Token 的文件提交到 Git。
- 若与现有工程存在依赖版本冲突，可在你的工程中单独调整 `package.json` 版本。
- API 行为以 GeeLark 官方文档为准；本包内 `docs/` 仅作需求与说明参考。

如有问题可联系发包同事。
