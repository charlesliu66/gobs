# Asset Library Guide — Game Director Video Skill

## Overview

The asset library is a folder containing all official game assets: screenshots, character renders,
key art, UI elements, gameplay footage, logos, maps, and sound files.

**Rule: Always check the asset library BEFORE generating anything with AI.**

### 默认素材库路径（Default Library Path）

当用户未指定素材库时，优先使用以下路径作为取材来源：

- **路径**：`\\10.21.100.89\Project Krad\GNG Assets Library`
- **用法**：分镜表中「使用到的素材名称」列：取自该库的写「素材库/子路径/文件名」或类型；缺的写「待补充」或「即梦/Kling 生成」。关键素材缺失时先列出缺镜，请用户优先提供，再确认是否 AI 生成。

## Critical Assets (Require Human Confirmation Before AI Generation)

These asset types MUST NOT be AI-generated without explicit user approval:

| Asset Type | Why It's Critical |
|-----------|------------------|
| Named characters | Brand identity, must match official art |
| World maps | Established lore, must be accurate |
| Game logo / wordmark | Legal trademark, zero tolerance for error |
| Key art / hero images | Used in official marketing materials |
| UI elements | Must match in-game visuals exactly |
| Mascots | Brand mascots have strict guidelines |

**When a critical asset is missing from the library:**
> ⚠️ "I couldn't find [asset_name] in the library. This is a critical brand asset. Please either:
> 1. Add it to the library at [suggested_path], or
> 2. Confirm I can AI-generate a stylized version (note: may not match official art)"

## 角色图用于视频生成时的「一致性」要求

- 同一角色在多镜头中出现时，若仅用**文生视频（T2V）**，每个镜头独立生成，**角色造型会不一致**。
- 若素材库中已有该角色的立绘/定妆图（如 `characters/浪人.png`），应用作 **图生视频（I2V）的首帧**，以保持多镜造型一致。
- 调用视频 API 时，I2V 模式需要传入**图片 URL**：必须是**API 服务器能通过 HTTP GET 直接拿到图片字节**的地址（不能是本地路径或 UNC，因为接口跑在云端）。  
  **可选**：上传至对象存储/图床得直链（推荐）；或尝试 Google Drive 直链（部分情况可用，稳定性不如对象存储）。批量生成时对该角色镜头使用 `asset_source: library:<该 URL>`。  
  详见技能内「角色一致性」与 `references/rules_and_formulas.md` 的「角色一致性与 I2V 首帧」。

## Non-Critical Assets (Can Be AI-Generated Automatically)

These can be generated by Kling AI without confirmation:

- Generic environment/background shots
- Crowd scenes, atmosphere shots
- Abstract VFX, particle effects
- Generic weapon effects
- Establishing shots without named characters
- Transition/interstitial shots

---

## How asset_manager.py Searches the Library

The script searches by:
1. **Filename match**: `hero_character.png` matches query "hero character"
2. **Parent folder match**: File in `characters/wizard/` matches query "wizard"
3. **Partial match**: Any query term appearing in path

**Query tips for the AI agent:**
- Be specific: `"main hero fullbody"` not just `"character"`
- Include context: `"forest environment screenshot"` not `"background"`
- For video clips: `"combat gameplay clip"` or `"cinematic cutscene"`

## Asset Source Column Values

In the production table, the "Asset Source" column uses these values:

| Value | Meaning |
|-------|---------|
| `Library: characters/hero.png` | Found in library, will use for image-to-video |
| `AI Generate` | Not in library, safe to generate automatically |
| `⚠️ Needs Confirmation: world_map` | Critical asset missing, user must decide |
