将 QAS 项目及依赖的 Skills 打包，拷贝到另一台电脑继续开发。

## 二、打包方式

### 方式 A：使用 PowerShell 脚本（推荐）

在 QAS 目录下打开 PowerShell，执行：

```powershell
.\scripts\package-for-transfer.ps1
```

会在 `QAS\QAS-package\` 下生成完整打包目录。

若脚本报编码错误，可尝试：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -Command "& { Set-Location 'C:\Users\wei.liu\Desktop\cursor_try\QAS'; .\scripts\package-for-transfer.ps1 }"
```

### 方式 B：手动打包

1. **新建目录** `QAS-package`，在其中建立子目录：
   - `QAS`
   - `video-pipeline`
   - `cursor-skills`
   - `config-templates`

2. **复制 QAS**：将整个 QAS 文件夹复制到 `QAS-package\QAS`，然后删除：
   - `QAS-package\QAS\node_modules`
   - `QAS-package\QAS\h5-video-tool\node_modules`
   - `QAS-package\QAS\h5-video-tool-api\node_modules`
   - `QAS-package\QAS\.git`（若不需要版本历史）
   - `QAS-package\QAS\*.env`（敏感信息，不要拷贝）
   - `QAS-package\QAS-package`（若有）

3. **复制 video-pipeline**：复制整个 video-pipeline 到 `QAS-package\video-pipeline`，删除其中的 `node_modules`

4. **复制 Skills**：将上述 5 个 skill 文件夹从 `%USERPROFILE%\.cursor\skills\` 复制到 `QAS-package\cursor-skills\`

5. **配置模板**：从 `QAS\scripts\DEPLOY-README-template.md` 复制到 `QAS-package\`，重命名为 `DEPLOY-README.md`；从 `QAS\config-templates\` 或 `QAS\config\geelark.json` 创建 `geelark.json.template`（脱敏后）放入 `QAS-package\config-templates\`

---

## 三、拷贝到新电脑

将整个 `QAS-package` 文件夹拷贝到 U 盘、网盘或局域网共享。

---

## 四、新电脑部署步骤

详见 `DEPLOY-README.md`（打包时生成）或 `scripts\DEPLOY-README-template.md`。

简要步骤：

1. 将 QAS 和 video-pipeline 放到同一父目录（如 `D:\dev\cursor_try\`）
2. 将 cursor-skills 下的 5 个文件夹复制到 `%USERPROFILE%\.cursor\skills\`
3. 填写 `config\geelark.json` 和 `h5-video-tool-api\.env`
4. 运行 `npm install` 安装各子项目依赖
5. 在 video-pipeline 目录运行 `npx playwright install chromium`
6. 创建 `Ai Videos` 输出目录（路径与 geelark.json 中 aiVideosPath 一致）

## 六、注意事项

- **不要拷贝**：`.env`、`gmail.env`、`config/geelark.json`（含真实密钥）到新电脑，应在新电脑重新填写
- **video-pipeline 路径**：skill 默认找 `Desktop\cursor_try\video-pipeline`，若路径不同需设环境变量 `VIDEO_PIPELINE_DIR`
- **Ai test 素材库**：若使用「按 prompt 自动匹配素材」功能，需在新电脑准备 `Ai test` 目录
