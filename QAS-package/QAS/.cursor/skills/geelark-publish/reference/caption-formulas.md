# 各平台易火文案与标签公式

基于视频的生成 prompt，按平台特性自动生成文案和标签建议。执行 geelark-publish 时，若用户未提供文案，**必须**读取本文档并按各平台公式生成建议。

---

## 通用原则

- **前 3 秒**：各平台首句/标题决定是否继续观看，必须抓眼球
- **互动感**：提问、邀评、邀分享可提升互动率
- **长度**：短视频适配短文案，避免过长

---

## TikTok

### 文案结构
- **长度**：30–80 字为佳
- **风格**：简洁、有趣、口语化、可带缩写（如 af、fr、ngl）
- **公式**：
  - 悬念式：结果前置 + 留白（「你绝对想不到…」「看完别哭」）
  - 互动式：「你敢…吗」「谁懂」「有没有同款」
  - 竞猜式：「猜猜最后发生了什么」「数数有几只」
  - 情感共鸣：「只有 xx 才懂」「破防了」

### 标签策略
- 3–5 个即可，避免堆砌
- 1 个泛流量：#fyp #foryou #viral
- 2–3 个垂直：内容相关，如 #游戏 #动漫 #剪辑
- 1 个长尾：具体场景或梗

### 示例（基于「血坦持武器奔跑」）
- 文案：「这速度你追得上？🔥」「谁懂，血坦跑起来根本拦不住」
- 标签：#fyp #gaming #anime #gameplay #shorts

---

## Instagram Reels

### 文案结构
- **Hook（前 125 字符）**：好奇、争议或直接点出人群
- **正文**：2–3 句，补足信息、情绪或背景
- **CTA**：明确行动（保存、评论、分享、@ 好友）

### 公式
- Hook：大胆说法 / 好奇缺口 / 对特定人群喊话
- 正文：解释、故事或情绪补充
- CTA：「双击保存」「评论区见」「@ 你的xx朋友」

### 标签策略
- 3–8 个
- 2–3 个大盘：#reels #viral #explore
- 3–5 个垂直：内容、风格、品类
- 2–3 个长尾：具体话题或 niche

### 示例（基于「血坦持武器奔跑」）
- 文案：「血坦开跑，谁都拦不住 🏃‍♂️💨 // 你最喜欢哪一帧？// 保存下来当壁纸」
- 标签：#reels #gaming #anime #gameart #art #shorts #videogames

---

## YouTube Shorts

### 标题
- **长度**：8–15 词或 55–60 字符，避免被截断
- **公式**：结果/钩子 + 具体主题 + 受众（可选）
- **风格**：可加 1–2 个 emoji，关键词放前面

### 描述
- 前两行放核心关键词
- 可加 #Shorts + 2–4 个垂直标签

### 示例（基于「血坦持武器奔跑」）
- 标题：「血坦全速奔跑，这画面绝了 🔥」
- 描述：「血坦持武器奔跑的 5 秒高光 #Shorts #Gaming #Anime #ShortForm」

---

## Facebook

### 文案结构
- **长度**：可略长，但前 1–2 句决定是否展开
- **风格**：偏故事、情感、分享型

### 公式
- 故事式：简短背景 + 转折/高潮
- 提问式：引发讨论或回忆
- 分享邀请：「分享给你的 xx 朋友」

### 示例（基于「血坦持武器奔跑」）
- 文案：「血坦持武器全力奔跑的瞬间，有没有勾起你的回忆？🔮 分享给也玩过这个角色的朋友」

## 生成指引（供 LLM 使用）

当用户提供**视频的生成 prompt**（如「血坦拿着武器奔跑的5秒视频」）时：

1. 提取**主体、动作、风格、时长**等要素
2. 按选定平台，套用上述公式生成 2–3 组文案 + 标签
3. 每组可略有差异（偏情感 / 偏互动 / 偏悬念）
4. 输出格式：按平台分块，每平台给出 caption 与 hashtags

**输出模板**：
```markdown
## 平台：TikTok
**文案 A**：...
**文案 B**：...
**标签**：#tag1 #tag2 #tag3

## 平台：Instagram Reels
...

（依此类推）
```
