# 评论自动生成 · 需求确认书（终稿）

**版本**：终稿（与产品确认一致）  
**关联能力**：在现有「批量评论 + 定时任务」流程上，增加**规则库**与**按条生成/刷新评论**；提交与调度逻辑与现网保持一致。

---

## 一、背景与目标

- 用户可**一次性上传并长期保存**「评论生成规则库」（纯文本、单库可含**多条**文本规则），后续**无需重复上传**，可直接选用。
- 用户输入**自然语言生成提示词**，并选择**规则库**、**输出语言**。
- 用户粘贴**设备、链接、时间**等批量任务信息；系统**解析成功后即触发批量生成**，为每条任务生成对应评论正文。
- **批量提交、定时任务**等下游流程与现有行为**保持一致**。

---

## 二、用户流程（概要）

1. **规则库管理（持久化）**  
   在批量评论相关界面提供：规则库的**新增、删除**及必要时的**列表展示/命名**（具体字段以 UI 稿为准）。上传成功后服务端持久保存，下次进入可直接选用。

2. **生成配置**  
   输入**生成提示词**；选择**一条已保存的规则库**；选择**输出语言**。

3. **批量任务与解析**  
   用户粘贴设备、链接、时间等信息（格式与现网批量评论一致）；系统解析得到 N 条评论任务。

4. **批量生成与单条刷新**  
   - **解析完成后**，系统自动对**每条任务**调用生成能力，将评论写入该条任务。  
   - 用户对某条评论不满意时，可点击该条评论旁的**「刷新」**按钮，**仅重新生成这一条**，不影响其他已生成内容（除非产品另有「全部重生成」需求，本期不包含）。

5. **提交与定时**  
   用户确认后批量提交；**定时任务、SeaTalk 等流程不变**。

## 三、规则库形态（已确认）

| 项 | 结论 |
|----|------|
| 内容形态 | **纯文本**（可支持换行分段表示多条规则，或 UI 内多条输入框，存储层面为「单库多条文本规则」） |
| 单库结构 | **一个规则库内包含多条文本规则** |

---

## 四、生成输入范围（已确认）

| 项 | 结论 |
|----|------|
| 链接侧信息 | **不需要**带入链接标题、页面摘要；**不需要**为生成评论做网页爬取。生成所依信息以**用户提示词 + 选中规则库 + 语言 + 任务行上下文（如设备/链接/时间等解析结果）**为准（若实现上仅传提示词+规则+语言亦可，需与开发对齐最小必要字段）。 |

---

## 五、异常与提示（已确认）

| 场景 | 结论 |
|------|------|
| 单条生成失败 | **跳过该条**（不阻塞整批）；**必须向用户明确提示**失败原因或「第 X 条生成失败」类信息，便于补刷或手改 |

---

## 六、批量生成时机（已确认）

| 项 | 结论 |
|----|------|
| 生成时机 | **解析成功后即批量生成**各条评论（非仅在最终提交前才生成） |

---

## 七、功能范围（本期）

| 包含 | 说明 |
|------|------|
| 规则库 | 纯文本、单库多条规则；持久化；界面新增/删除与选用 |
| 生成 | 提示词、规则库、语言；解析后批量生成；单条刷新按钮 |
| 批量评论页 | 与现有粘贴解析、提交定时流程集成 |
| 交互 | 单条不满意 → 刷新重生成该条 |

| 不包含（除非另开需求） | 示例 |
|------------------------|------|
| 网页抓取、标题/摘要 | 已明确不需要 |
| 批量「全部重新生成」按钮 | 本期未要求；仅单条刷新 |

---

## 八、验收要点（建议）

1. 规则库上传后持久存在，可删除；单库可维护多条文本规则。  
2. 解析成功后自动为每条任务生成评论；失败条跳过并有**用户可见提示**。  
3. 每条评论旁有**刷新**，仅重生成该条。  
4. 提交后定时任务行为与改动前一致。

## 九、待开发阶段细化（非阻塞确认）

- 规则库在存储层的表结构/文件结构（单库多条文本的存储方式）。  
- 生成服务对接的模型与配额、超时与重试策略。  
- 「跳过该条」时该条任务在 UI 上展示为空白、占位文案还是可编辑空状态（建议：可编辑 + 提示失败，便于用户手填或点刷新）。
