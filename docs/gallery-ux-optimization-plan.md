# 「我的成片」页面 UX 优化方案

> 文件：`h5-video-tool/src/components/GalleryView.tsx` + `h5-video-tool/src/pages/Gallery.tsx`
> 状态：待确认
> 创建：2026-04-15

---

## 一、现状问题诊断

通过代码阅读 + 截图分析，当前"我的成片"存在以下 UX 问题：

### P0 — 信息噪音太多，主要内容被淹没

| 位置 | 现有文字（节选） | 问题 |
|---|---|---|
| 页面顶部段落 | "此处只收录通过本站调用本 API 生成…即梦 App、网页、剪映里生成的不会自动同步…" | 用户进来就被一堆限制说明迎头浇下，首眼看不到自己的视频 |
| 「服务端 output」tab 内 | "和「本机历史」的区别"4条 bullet + 另一段 VITE_API_BASE_URL 说明 | 技术细节不是普通用户需要的，且在空状态时占满整个视图 |
| 「本机历史」空状态 | "若只在即梦/剪映客户端里生成…此处不会有记录…见下方服务端 output 说明手动放入文件夹" | 空状态应该是 CTA，而不是继续解释 |
| 「服务端 output」空状态 | 4 段文字 + 多个 `<code>` 路径说明 | 普通用户看不懂，技术用户也可以直接去服务器看 |

### P1 — 两个页面 + 三个 Tab，成片分布散乱

```
/gallery（我的成片）
  ├── 本机历史
  └── 服务端 output 成片

/history（历史）
  ├── 可灵云端
  ├── 本机历史    ← 与 gallery 重叠
  └── 批量任务看板
```

用户不清楚"本机历史去哪里看"——两个页面都有。

### P2 — 视频卡片视觉密度低

- 列表布局：一行只能看一个视频，需要大量滚动
- 文件路径（`output/admin/dreamina_c426c0cd...mp4`）暴露给用户，无意义
- "加入本机历史"按钮概念不直观（用户不理解"本机历史"是什么）

---

## 二、优化目标

> **一句话目标**：用户进入"我的成片"，第一眼就看到所有成片缩略图，想看哪个点哪个，没有任何需要阅读的文字。

---

## 三、具体改动方案

### 改动 A：删除顶部说明段落（高优先级）

**Before（约 4 行解释）：**
```
此处只收录通过本站（Studio / 高级制片）调用本 API 生成并写入浏览器或服务端 output 的成片。
在即梦 App、即梦网页、剪映里单独生成的视频不会自动同步…
```

**After：**
直接删除。页面副标题"本地生成记录与服务端近期输出"已足够。

---

### 改动 B：重命名 Tab，简化标签含义

| 旧标签 | 新标签 | 说明 |
|---|---|---|
| `本机历史 (N)` | `我的成片 (N)` | 更直白，不用理解"本机历史"概念 |
| `服务端 output 成片` | `服务端文件` | 去掉技术味的"output"，名字更简洁 |

Tab 旁边加一个 `?` 图标 tooltip，鼠标悬浮才展示简短说明（一行内解释清楚），而不是直接展开。

---

### 改动 C：「服务端文件」tab 内移除说明 Block

**Before：**
- 整个"和「本机历史」的区别"4条 bullet box
- 下方另有一段 VITE_API_BASE_URL 说明

**After：**
- 完全删除这两段
- 保留「刷新列表」按钮
- 若空状态，展示：

```
[图标] 服务端暂无视频文件

[刷新列表]  （小字：在「我的成片」里找不到？去 Studio 生成一个）
```

---

### 改动 D：空状态改为简洁 CTA

**「我的成片」tab 空状态 Before：**
> "在「创作」里一键生成…若只在即梦/剪映客户端里生成…请在本站发起生成，或见下方…"

**After：**
```
[大图标]

还没有成片

[去 Studio 创作]   [高级制片]
```
取消所有条件说明，直接给出两个行动按钮。

---

### 改动 E：视频卡片视觉优化

**文件路径处理：**
- 原来显示完整路径：`output/admin/dreamina_c426c0cd2dfb_1776229192704.mp4`
- 改为只显示智能文件名：去掉路径前缀，显示不含 hash 的可读名称（如"dreamina 2026/4/15 12:59"）

**按钮文案优化：**
- `加入本机历史` → `保存到我的成片` （更直白）

**可选：网格视图（P2 优先级）**
- 若成片数量 ≥ 4，提供一个切换按钮在"列表视图"和"网格视图"之间切换
- 网格视图：3列，点击卡片弹出大图预览 + 下载/分发操作

---

### 改动 F：「批量任务 & 云端历史 →」入口优化（可选）

当前右上角按钮容易被忽视。建议：
- 如果用户是从"服务端文件"tab 进来、且结果为空，在空状态底部展示："找不到？ → 查看批量任务和可灵云端" 的引导链接
- 把右上角按钮文案改为 `历史记录 →` 更简洁

---

## 四、不改动范围

以下不在本次优化范围内，保持现有逻辑：
- `GalleryView` 与 `History.tsx` 的页面分拆结构（属于架构问题，可后续单独规划合并）
- 后端 API 接口
- 视频播放、下载、星标、删除功能逻辑
