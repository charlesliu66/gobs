# 全部配置清单（GeeLark + SeaTalk + 任务通知 + Vercel）

按顺序完成即可。**真实密钥只写在环境变量里，不要提交到 Git。**

---

## 一、本地开发（`web/.env.local`）

### 1. 一键生成模板与 CRON_SECRET

在 **`web`** 目录执行：

```bash
npm run setup:env
```

- 若没有 `.env.local`，会从 `.env.local.example` 复制并创建。
- 会自动追加随机 **`CRON_SECRET`**（若尚未配置）。

### 2. 手动编辑 `web/.env.local`（必填项）

| 变量 | 说明 |
|------|------|
| `GEELARK_BEARER_TOKEN` | GeeLark 开放平台 Bearer Token |
| `GEELARK_BASE_URL` | 默认 `https://openapi.geelark.com/open/v1` |
| `GEELARK_FB_COMMENT_PATH` | （可选）Facebook 评论 RPA 路径；默认 `rpa/task/facebookRandomComment`，与控制台 OpenAPI 不一致时再改 |
| `SEATALK_APP_ID` | SeaTalk 开放平台 App ID |
| `SEATALK_APP_SECRET` | SeaTalk 开放平台 App Secret |
| `SEATALK_EMPLOYEE_CODE` | **推荐**：你的 SeaTalk **employee_code**，消息发到与机器人的 **单聊**（owner）。若同时配置了 `SEATALK_GROUP_ID`，**优先单聊** |
| `SEATALK_GROUP_ID` | 可选：仅当**未配置** `SEATALK_EMPLOYEE_CODE` 时发群；需机器人已在群内，获取方式见 `docs/SeaTalk获取group_id.md` |
| `SEATALK_SIGNING_SECRET` | （可选但**强烈建议**）Event Callback 页的 **Signing Secret**；回调 URL 填 `https://你的域名/api/seatalk/callback`，详见 `docs/SeaTalk事件回调配置.md` |

**二选一（发消息方式）：**

- **推荐 Open API 单聊 owner**：`SEATALK_APP_ID` + `SEATALK_APP_SECRET` + `SEATALK_EMPLOYEE_CODE`（可不配群）。
- **或 Webhook**：配置 `SEATALK_WEBHOOK_URL`（则优先走 Webhook，不走 App ID）。

### 3. 任务状态轮询（启动/完成/失败 推 SeaTalk）

云端 **必须** 有 Redis 类持久化（Vercel 无本地磁盘）：

**任选其一：**

**A）Vercel KV（推荐，与 Vercel 同控制台）**

1. Vercel 项目 → Storage → Create → KV  
2. 绑定到本项目后，会自动注入：

   - `KV_REST_API_URL`
   - `KV_REST_API_TOKEN`

**B）Upstash Redis**

1. [Upstash Console](https://console.upstash.com/) 创建 Redis → REST API  
2. 复制：

   - `UPSTASH_REDIS_REST_URL`
   - `UPSTASH_REDIS_REST_TOKEN`

### 4. 本地文件状态（仅本地长期跑 `npm start` 时）

不配 Redis 时，会使用 `web/.data/seatalk-task-state.json`（已 gitignore）。  
`npm run dev` 也可用，但重启后行为以实际为准。

### 5. AI 生成评论与规则库（可选）

批量评论页「规则库 + 提示词」生成、`/api/comment-rules/generate` 等需要 **至少一种** LLM 密钥。

| 变量 | 说明 |
|------|------|
| `GEMINI_API_KEY` 或 `GOOGLE_API_KEY` | **优先**：Google Gemini（`AIza...`） |
| `GEMINI_MODEL` | 可选；不配则服务端按可用型号回退 |
| `OPENAI_API_KEY` | 未配置 Gemini 时使用 OpenAI 兼容接口 |
| `OPENAI_API_BASE` | 可选，默认官方 `https://api.openai.com/v1` |
| `OPENAI_MODEL` | 可选，默认如 `gpt-4o-mini` |
| `COMMENT_COMMANDER_PY_PATH` | 可选：全局 Safety Filter 的 `commander.py` **绝对或相对路径**（相对 `process.cwd()`）；不配则在仓库根、`web/` 等路径自动查找，见 `commander.py.example` |

**规则库持久化**：与任务状态相同，优先 **Vercel KV / Upstash**（`KV_REST_*` 或 `UPSTASH_*`）；否则本地 `web/.data/comment-rule-libraries.json`（**生产环境务必配置 KV**）。

**自检**：浏览器打开 **`/api/ai-config-status`**（如 `http://localhost:3000/api/ai-config-status`），`ready: true` 表示至少检测到一种可用 AI 密钥。  
实现说明见 **`docs/评论自动生成-实现说明.md`**。

## 二、Vercel 线上环境

在项目 **Settings → Environment Variables** 添加上述**全部**变量（与本地一致），至少包括：

- `GEELARK_BEARER_TOKEN`
- `SEATALK_APP_ID`、`SEATALK_APP_SECRET`、**`SEATALK_EMPLOYEE_CODE`（推荐）或 `SEATALK_GROUP_ID`**、`SEATALK_SIGNING_SECRET`（若启用事件回调）
- `CRON_SECRET`（与本地 `setup:env` 生成值**相同或另设一串**，与 Cron 校验一致）
- `KV_REST_API_URL`、`KV_REST_API_TOKEN`（或 Upstash 两项）
- 若使用 **AI 规则库生成评论**：`GEMINI_API_KEY` / `GOOGLE_API_KEY` 与/或 `OPENAI_API_KEY`，以及可选 `COMMENT_COMMANDER_PY_PATH`（若用全局 `commander.py`）

**Cron**：仓库内已有 `web/vercel.json`（每 2 分钟调用 `/api/cron/task-seatalk-sync`）。  
部署后请在 Vercel → Cron Jobs 确认已启用；若套餐限制频率，可改 `vercel.json` 中的 `schedule`。

**手动测同步：**

```text
https://你的域名/api/cron/task-seatalk-sync?secret=你的CRON_SECRET
```

## 三、SeaTalk 开放平台检查项

- 应用为 **Bot**，状态 **Online**  
- **单聊 owner（推荐）**：你已添加/与机器人有过会话，`SEATALK_EMPLOYEE_CODE` 与后台一致  
- **或发群**：机器人在目标 **群内**，`SEATALK_GROUP_ID` 与控制台一致（仅当未配 `SEATALK_EMPLOYEE_CODE` 时生效）  
- 已开权限：**发消息**、**Get Group Info**（若用群信息接口）

## 六、收不到 SeaTalk 消息时

1. **KV 只存任务状态，不能代替 SeaTalk**：必须单独配置 `SEATALK_APP_ID` / `SEATALK_APP_SECRET` / **`SEATALK_EMPLOYEE_CODE`（推荐）或 `SEATALK_GROUP_ID`**（或 Webhook），且 **Vercel 环境变量里也要有**，与本地一致。  
2. 浏览器打开 **`/api/seatalk/status`**（如 `https://你的域名/api/seatalk/status`）：  
   - `ready: true` 才表示发消息所需变量齐全；  
   - `ready: false` 看 `hint` 说明缺什么。  
3. 提交批量评论后，接口 JSON 里会有 **`seatalk`** 字段：  
   - `{"ok":true}` 表示已调用 SeaTalk 成功；  
   - `skipped` 表示未配置；  
   - `error` 为开放平台返回的错误信息（便于排查）。
