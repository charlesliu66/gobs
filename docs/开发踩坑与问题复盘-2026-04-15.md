## 一、鉴权与访问控制

### 1.1 视频播放黑屏 403

| 项目 | 内容 |
|------|------|
| 关联 commit | `b7cd781`（引入） → `671b02c`（修复） |
| 现象 | `<video>` 标签播放视频时黑屏，显示 0:00 |
| 根因 | 在 `auth.ts` 中为 `/api/video/file` 添加鉴权豁免，允许 `<video>` 标签无 JWT 访问，但没有同步检查 `video.ts` 路由内部的 `req.user?.username` 二次鉴权逻辑。无 JWT 时 username 变成 `_default`，而视频文件在 `output/admin/` 下，目录不符返回 403 |
| 修复 | 路由内部改为从 query 参数提取用户，不再依赖 `req.user` |
| 教训 | **改 auth bypass 时必须同步检查路由内部所有 `req.user` 引用** |

## 核心经验总结

1. **鉴权改动要两端联查** — auth 中间件 bypass 和路由内部 `req.user` 必须同步修改
2. **数据清理用白名单** — 绝不能用全量模式匹配删除，必须明确列出可清理的字段名
3. **外部 API 并发要保守** — 先从 1 开始，逐步上调，别一上来就开 5 个槽
4. **服务器配置不能想当然** — Nginx 超时、上传限制、端口绑定都要逐项检查
5. **路径不要硬编码** — 本地 Windows 和服务器 Linux 路径必然不同，一律走 `.env`
6. **持久化要考虑刷新** — 任何异步任务的 ID 都必须持久化，刷新后能恢复
7. **巨石文件及早拆分** — 超过 1000 行的文件应该拆分，否则人和 AI 都容易改错
8. **多 Agent 协作要有协议** — 文件所有权、命名前缀、合入顺序，提前约定
