# H5「爆款视频分析」功能设计（已定稿项）

## 1. 背景与目标

在 H5 中新增「爆款视频分析」：用户提交 **抖音 / TikTok 系视频链接**（第一版主战场），系统输出可下载素材、字幕与翻译、深度分析、脚本二创建议、AI 分镜表，并与现有创作流（Studio / 分镜）可衔接。

## 2. 已确认决策

| 决策点 | 选择 | 说明 |
|--------|------|------|
| 链接平台 | **A：抖音 / TikTok 系** | 下载与解析以 yt-dlp 类能力为主；需容忍 extractor 失效与合规边界 |
| 站外数据指标 | **① 仅视听 + 文案** | **不展示**平台官方播放/点赞/转发等精确数；深度分析以 **视频内容 + 转写文本 + LLM** 为主；凡基于画面/节奏的推断须标注 **「推断」** 或 **「内容侧评估」**，避免误导为真实后台数据 |

## 3. 功能分块（五 Tab）

1. **素材下载**：在合法与技术可行范围内提供 **视频 / 音频 / 封面** 下载链接或打包；失败时给出原因与建议（换链、本地上传等）。
2. **字幕 & 翻译**：以 **音轨 ASR** 为主生成时间轴字幕；目标语言翻译；为后续 Tab 提供统一文本输入。
3. **深度分析**：**不含**真实站外指标。包含：可计算的 **内容侧** 指标（如时长、估算节奏/切点密度、前三秒画面与文案钩子分析）、**爆款结构**与**转场/剪辑**的归纳、**爆款原因**以 LLM 结构化输出并带免责声明。
4. **脚本二创**：基于字幕与结构摘要，输出 **改编方向、新脚本建议、风险提示**（版权与原创）。
5. **AI 分镜**：输出 **时间轴 + 镜头描述 + 生成式 prompt + 运镜/灯光建议**（对齐产品原型表格）；支持 **复制** 与后续 **导入 Studio**（实现阶段再定格式）。

## 4. 数据流（逻辑顺序）

```
链接/文件 → 拉取媒体（视频+音+封面）→ ASR/字幕 → 并行：
  · Tab1 下载物
  · Tab2 字幕翻译
  · Tab3 深度分析（视听特征 + LLM，不依赖站外指标）
  · Tab4 二创脚本（依赖字幕+Tab3 摘要）
  · Tab5 分镜表（依赖字幕+Tab3+风格约束）
```

## 5. 架构原则（实现时遵循）

- **后端**：异步任务（链接解析与下载耗时）、统一 job id、结果落盘或对象存储、API 轮询或 SSE/WebSocket 推送进度。
- **工具链**：与仓库内 **Viral_Agent**（yt-dlp、ffmpeg）思路对齐；**AudioAgent** 负责配乐，与本功能 **解耦**，仅在「分析后回 Studio 再配乐」时衔接。
- **合规**：用户协议中明确 **仅分析用户有权使用的链接**、下载物用途；LLM 输出标注 **非平台官方数据**。

## 6. MVP 建议切分

- **MVP1**：抖音/TikTok 链接 → 下载三件套 + ASR 字幕 + Tab3 轻量深度分析（钩子 + 结构）+ Tab5 简化分镜表。
- **MVP2**：翻译 + Tab4 二创 + 导出与进 Studio。
- **MVP3**：更强视听算法（切点、节奏）与 UI  polish。

## 7. 未决事项（实现前再定）

- 任务队列与存储部署形态（单机 / Docker / 云）。
- ASR 与 LLM 供应商（是否复用现有 `h5-video-tool-api` 的 Gemini 等）。
- 是否提供 **本地上传** 作为链接失败时的 fallback（建议有）。

## 8. 下一步

经产品确认本设计后，使用 **writing-plans** 技能输出分阶段 **实现计划**（路由、API、前端 Tab、环境变量与依赖清单）。
