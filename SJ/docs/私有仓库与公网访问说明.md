# 私有仓库 + 有链接即可使用（Token 已写在代码里）

## 你的目标

- GitHub 仓库为 **Private**（代码不公开）。
- **Token 已写在** `web/lib/geelark-token.ts`，Vercel 上**可不再单独配置** `GEELARK_BEARER_TOKEN`。
- **有链接的人**：在浏览器打开公网地址即可用，**不需要**自己部署、装 Node、配 `.env`。

> 说明：**部署上线**仍需要你（或管理员）在 Vercel **连接仓库并点一次 Deploy**；之后同事、客户只保存/打开 `https://xxx.vercel.app` 即可。

---

## 操作步骤（你本人做一次）

### 1. GitHub 建「私有」仓库

- New repository → 勾选 **Private** → 把本项目 `push` 上去。

### 2. Vercel 部署（无需在 Vercel 填环境变量）

1. 打开 [vercel.com/new](https://vercel.com/new)，用 GitHub 授权，**Import** 该私有仓库。
2. **Root Directory** 选 **`web`**（若仓库根目录就是 `web` 内容则不用改）。
3. **不要**在 Environment Variables 里再配 `GEELARK_BEARER_TOKEN`（已写在代码里）。
4. 点 **Deploy**，完成后复制 **`https://你的项目.vercel.app`**。

### 3. 把链接发给需要的人

对方：**只打开这个链接**，无需任何安装或部署。

---

## 安全提示

| 项目 | 说明 |
|------|------|
| 私有仓库 | 代码与 Token 只对有仓库权限的账号可见。 |
| 仍勿公开仓库 | 若误设为 Public，Token 会泄露。 |
| 链接即能力 | 任何拿到 Vercel 链接的人都能用你的 GeeLark 额度，重要场景建议加登录或访问控制。 |

---

## 若以后要改 Token

改 `web/lib/geelark-token.ts` 里的字符串 → `git push` → Vercel 自动重新部署。
