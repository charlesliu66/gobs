# H5 剪辑工作台 — 设计说明（强 Agent + 导出 MP4 + 画幅可选）

**日期**: 2026-03-31  
**状态**: 已确认方向  
**关联**: `2025-03-12-h5-frontend-prd.md`（现有生成流水线）、`2026-03-31-h5-editor-implementation-plan.md`（实现计划）

---

## 1. 已确认目标

| 项 | 决策 |
|----|------|
| 技术路径 | **A**：浏览器内预览级时间轴 + **服务端 FFmpeg（或等价）合成** 一键导出 MP4 |
| 交互策略 | **强 Agent**：在生成质量足够高的前提下，**尽量减少用户自定义操作**；时间轴以「Agent 生成 + 用户少量确认/微调」为主 |
| 成片 | **必须**支持「一键导出 MP4」（下载到本地或返回可下载 URL） |
| 画幅 | **用户可选择**（如 9:16 / 16:9 / 1:1 / 4:3 等），并贯穿预览安全区与导出分辨率 |

---

## 2. 产品原则（强 Agent）

1. **默认少问**：用户用自然语言描述意图（风格、时长感、情绪、禁区）；Agent 自动完成素材理解、BGM 方案、初剪排布。
2. **可反悔**：重大步骤提供「应用 / 撤销 / 再生成一版」，避免一次写死时间轴。
3. **工程可复现**：所有 Agent 输出必须落到**版本化的时间轴 JSON**（见 §4），导出与预览同源，禁止「只有 UI 状态、无法序列化」。
4. **质量闸门**：自动混剪后提供 **预览**；导出前可选「快速质检」提示（时长、黑边、音量突变），不强制阻塞 MVP。

---

## 3. 能力分层

### 3.1 素材与打标（异步）

- 上传素材 → 任务队列 → 视觉/多模态理解（具体模型/API 可插拔）→ 存储 **per-clip 标签 + 可选时间片段级标注**（便于按「高潮/人物/场景」选镜）。
- 左侧素材库展示缩略图、时长、标签摘要；Agent 与用户均可按标签检索。

### 3.2 BGM

- 输入：风格、情绪、目标总时长（与成片一致）、可选参考 BPM 范围。
- 输出：音频文件 URL + 元数据（时长、若可得的节拍/段落点）。
- 强 Agent 场景下：**由 Agent 发起生成**，用户仅需在对话中确认或一句话修改。

### 3.3 混剪

- 输入：标签/片段候选 + BGM 结构（段落或节拍点）+ 画幅（影响裁剪策略）。
- 输出：时间轴 JSON（多轨：视频片段 in/out、音轨、可选字幕轨占位）。
- Agent 负责生成 **初版方案**；用户可仅在时间轴上微调 in/out 或替换某一段素材。

### 3.4 导出 MP4

- 前端提交：`projectId` + 冻结的时间轴版本 + `aspectRatio` + 编码预设（可默认）。
- 后端：拉取素材与 BGM → FFmpeg 合成 → 上传对象存储或临时文件 → 返回 `downloadUrl` / 轮询 `jobId`。
- 画幅：导出时统一 scale/crop/pad 到目标分辨率（与预览一致，避免「预览与成片不一致」）。

## 4. 核心数据：时间轴 Schema（概念）

以下为逻辑结构，具体字段在实现阶段以 TypeScript 类型 + API 契约束定：

- `project`: `{ id, aspectRatio, fps, duration }`
- `tracks[]`: `{ type: 'video'|'audio', id, clips[] }`
- `clip`（视频）：`{ assetId, sourceStart, sourceEnd, timelineStart, transform? }`
- `clip`（音频）：`{ assetId, sourceStart, sourceEnd, timelineStart, gainDb? }`
- `asset`: `{ id, url, kind, tags[], meta }`
- `agentRevision`: `{ id, parentRevisionId?, summary, createdAt }`（便于撤销/对比）

预览与导出 **必须** 从同一 schema 渲染，避免双轨逻辑。

## 6. 风险与依赖

- **FFmpeg 运行环境**：需 Node 服务可调用 `ffmpeg` 或使用云转码；部署文档需补充。
- **第三方能力**：打标、音乐生成、LLM 均可能涉及 API Key 与合规；建议抽象为 `providers/*`，便于替换。
- **大文件与时长**：上传与合成任务需超时、重试、进度查询；避免浏览器阻塞。
