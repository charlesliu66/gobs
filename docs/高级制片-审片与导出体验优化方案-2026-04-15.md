# 高级制片 · 审片与导出体验优化方案

> 创建日期：2026-04-15  
> 背景：当前导出页（Step 5）以静态网格展示分镜卡片，用户无法感受叙事节奏、无法连续审片、无法管理多版本，导致"审片→迭代→导出"链路断裂。  
> 本文档描述三个层次的体验优化，并给出实施优先级。

---

## 核心用户旅程（As-Is vs To-Be）

### 现状（As-Is）痛点

```
[分镜表] → [逐镜生成视频] → [导出页：静态网格]
                                      ↓
              用户只能逐个点开视频，无法感受整体节奏
              不满意某镜 → 必须手动返回 Step 4 重新找到那一镜
              每次重新生成会覆盖原版本，没有退路
              配音/精修 → 必须另外到剪辑工作台，数据需手动导入
```

### 目标（To-Be）

```
[分镜表] → [逐镜生成视频] → [放映室：连续审片]
                                      ↓
              一键播放所有镜头串联预览，分镜条高亮当前镜
              发现问题镜 → 点击分镜条 → 右侧浮出「重新生成」操作
              多次生成保留版本历史，可在分镜条切换 v1/v2/v3
              满意后 → 「在剪辑器中打开」→ 自动导入所有镜头，继续精修配音
```

---

## 优化层次与优先级

| 优先级 | 层次 | 名称 | 工作量估算 | 核心价值 |
|:---:|---|---|---|---|
| **P0** | 第一层 | 放映室连续播放器 | 3-5 天 | 让用户第一次能"看到"自己的片子 |
| **P1** | 第二层 | 镜头版本历史 | 5-7 天 | 让迭代有退路，不再怕生成覆盖 |
| **P2** | 第三层 | 「在剪辑器中打开」直通 | 3-4 天 | 打通两个工作台，复用现有编辑能力 |

---

## P0 · 放映室连续播放器

### 目标

在导出页顶部新增一个主播放区，将当前项目所有分镜视频（按顺序）自动串联连续播放，底部附分镜胶片条用于跳转。

### 布局草图

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                   主播放区（16:9）                       │
│               当前：镜3 / 共8镜  [◀] [▶] [⏸]           │
│                                                         │
└─────────────────────────────────────────────────────────┘
│ [镜1]  │ [镜2]  │ [镜3]  │ [镜4]  │ [镜5]  │ [镜6]  │
│  ✅ v2  │  ✅ v1  │ 🔵当前  │  ⏳生成中│  ⚪未生成│  ✅ v1  │
└─────────────────────────────────────────────────────────┘

下方保留原有分镜卡片网格（供细节查看）
```

### 交互规则

- **自动串联**：将所有 `vSrc !== null` 的镜头按 `shotIndex` 顺序串联播放
- **无视频镜头**：用静帧图占位，停留 2 秒后自动跳下一镜（或可配置）
- **分镜条状态标记**：
  - ✅ 绿色：已有视频
  - ⏳ 黄色：生成中
  - ⚪ 灰色：未生成
  - 🔵 蓝色高亮：当前播放镜
- **点击分镜条**：跳转到对应镜头，主播放区切换播放
- **播放控制**：上一镜 / 下一镜 / 暂停 / 全屏

### 实现要点

- 前端纯组件，无需后端改动
- 播放控制通过管理 `currentShotIndex` 状态 + `<video>` 的 `onEnded` 事件驱动自动切换
- 分镜条复用现有 `resolveVideoSrc` / `previewStillDataUrl` 数据

## P1 · 镜头版本历史

### 目标

每次生成不再覆盖原视频，而是追加为新版本（v1、v2、v3…）。用户可在分镜条或卡片上切换版本，选中版本参与串联播放和最终导出。

### 数据结构变更

```typescript
// 现有
interface ProductionShot {
  previewVideoPath?: string;   // 单条，每次生成覆盖
}

// 目标
interface ProductionShot {
  videoVersions: VideoVersion[];       // 多版本
  selectedVersionIndex: number;        // 当前选中版本（参与播放/导出）
}

interface VideoVersion {
  versionLabel: string;   // "v1" / "v2" ...
  path: string;           // 服务端文件路径
  generatedAt: string;    // ISO 时间戳
  model: string;          // 生成模型（即梦/VEO 等）
}
```

### 前端 UI 变更

- 分镜卡片右上角增加版本选择器：`v1 ▾`，下拉列表显示所有版本及生成时间
- 切换版本后，主播放器和卡片预览同步更新
- 分镜条小图标显示当前版本号

### 后端变更

- `POST /api/studio/shots/:id/generate`：不覆盖，追加到 `videoVersions[]`
- `PATCH /api/studio/shots/:id/version`：切换选中版本
- 存储：服务端按 `shot_<id>_v<n>_<timestamp>.mp4` 命名文件，不删除旧文件

### 注意事项

- 版本数建议上限 5 个，超过后提示用户清理（避免磁盘爆满）
- 兼容旧数据：现有 `previewVideoPath` 迁移为 `videoVersions[0]`

## P2 · 「在剪辑器中打开」直通

### 目标

用户在审片满意后，点击一个按钮，自动将当前所有分镜的「选中版本视频」按顺序导入到剪辑工作台的时间轴，无需手动上传。

### 交互流程

```
导出页 [在剪辑器中打开 →]
    ↓
弹出确认框：
  「将 6 个分镜视频（共约 42 秒）导入剪辑器？」
  [确认] [取消]
    ↓
自动创建新剪辑项目，名称继承自当前高级制片项目
    ↓
跳转到 /editor?project=xxx，时间轴已排好所有镜头片段
    ↓
用户在剪辑器中添加配音、BGM、字幕、调速、转场
```

### 实现要点

- 新增 API：`POST /api/editor/projects/from-production`
  - 入参：`productionProjectId`
  - 服务端将分镜视频 path 映射为剪辑器 Clip 对象，创建 Editor Project
  - 返回新建 `editorProjectId`
- 前端：`/editor?project=<id>` 打开时自动加载该项目
- 视频文件无需复制，剪辑器直接引用原始 path

### 价值说明

- 完全复用现有剪辑工作台的所有能力（配音、BGM、转场、字幕、导出）
- 分镜工作台专注"创意生成"，剪辑工作台专注"精修输出"，职责清晰
- 无需在分镜工作台内重复建设 Timeline 编辑器

---

## 不做的事（明确排除）

| 排除项 | 原因 |
|---|---|
| 在分镜工作台内置完整 Timeline 编辑器 | 剪辑器已有，重复造轮子；分镜阶段用户还在创意模式，Timeline 认知成本高 |
| 分镜页内置配音录制 | 放在剪辑器做更合适；分镜配音在最终精修前意义不大 |
| 自动合并导出（在分镜页直接 FFmpeg 拼接） | P0 阶段不必要；用户更需要的是先"看懂"自己的片子，再决定是否精修 |
