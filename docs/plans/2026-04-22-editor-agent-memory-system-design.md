# Editor Agent Memory System Design

更新时间：2026-04-22

## 目标

为剪辑 Agent 建立一套面向项目协作的记忆系统，不再只依赖当前页面的临时对话态，而是逐步形成：

1. `项目级记忆`：记住同一个项目里的事实、剪辑偏好、已确认卖点和禁用项
2. `用户级记忆`：记住这个用户与平台沟通的方式画像，并跨项目复用
3. `总结闭环`：在一段真实使用周期后，自动输出平台优化建议，指导产品持续迭代

设计目标不是“保存更多聊天记录”，而是“只保留会提升下一次协作质量的记忆”。

---

## 一、为什么现在要做

当前剪辑 Agent 的历史主要还是页面级状态：

- 刷新后容易丢失
- 切项目后不具备长期连续性
- 无法真正形成“这个项目一直在做什么”的上下文
- 无法形成“这个用户喜欢怎么和平台协作”的稳定画像

这会导致两个直接问题：

1. Agent 每次都像第一次接手这个项目
2. 平台即使收集了大量用户交互，也很难沉淀成后续优化依据

---

## 二、设计原则

### 1. 分层记忆，而不是全文长期保留

原始聊天不应该长期全量喂给模型。越往后越应该把它沉淀为结构化偏好和项目事实。

### 2. 项目记忆和用户记忆严格分离

项目记忆回答“这个项目要做什么”；用户记忆回答“这个用户喜欢怎么合作”。

### 3. 负向偏好比正向偏好更重要

“不要这种开头”“不要太长解释”“不要慢节奏 BGM”这类信息，优先级要高于泛泛的喜欢项。

### 4. 只有高价值信号才能升级成长期记忆

一次性吐槽、临时犹豫、处理中日志，不应该进入长期记忆。只有被重复表达或被行为验证的偏好，才应长期保留。

### 5. 记忆必须可解释、可编辑、可撤销

平台后续一定要允许用户看到“系统记住了什么”，否则记忆会变成不可控黑盒。

---

## 三、记忆分层模型

### L0：当前会话上下文

用途：

- 保证当前这次编辑任务的连续性
- 让 Agent 能理解刚刚 10 分钟内说过的话

建议内容：

- 最近 `8-12` 轮原始对话
- 当前一次创意 brief
- 当前时间轴状态摘要
- 当前一次未完成任务或未解决问题

特点：

- 不做长期存储依赖
- 每次请求模型时直接拼接
- 超出窗口后进入压缩流程

### L1：项目级结构化记忆

用途：

- 记住这个项目持续稳定的事实和创意偏好

建议字段：

- `projectGoal`：TikTok 内容 / 买量 / 混合
- `targetAudience`
- `coreSellingPoints[]`
- `doNotUse[]`
- `preferredHookStyle`
- `preferredSubtitleStyle`
- `preferredRhythm`
- `preferredBgmStyle`
- `ctaPreference`
- `confirmedCreativeDirections[]`
- `rejectedCreativeDirections[]`
- `importantTimelineDecisions[]`

特点：

- 生命周期跟随项目
- 可在不同会话中复用
- 是项目协作的主记忆层

### L2：用户级沟通方式画像

用途：

- 跨项目复用这个用户的协作方式偏好
- 让 Agent 的回应风格、确认方式、自动化强度更贴合用户习惯

建议字段：

- `responseStyle`：简短直接 / 简短带理由 / 详细解释
- `collaborationMode`：先做后说 / 先给方案再确认 / 强确认型
- `executionTolerance`：接受快速试错 / 偏好一次到位
- `controlPreference`：自动执行优先 / 建议优先 / 强控制
- `feedbackPattern`：模糊否定 / 具体纠错 / 指标导向
- `pacePreference`：快推进 / 稳推进
- `platformLanguageStyle`：偏业务词 / 偏创意词 / 偏执行词
- `negativePreferences[]`

特点：

- 这是用户级长期记忆
- 不能跟某一个项目事实混存
- 每隔一段时间重算，避免画像僵化

### L3：周期总结与优化洞察

用途：

- 面向产品和平台层，不直接作为单次 Agent 长上下文
- 从真实用户体验中抽取“平台该怎么继续优化”

建议内容：

- 本周期最常见的 Agent 误判
- 本周期最常见的返工原因
- 被采纳最多的 Hook / 字幕 / 节奏策略
- 用户最常重复表达的偏好
- 哪类用户更适合强自动化，哪类更适合半自动协作
- 记忆系统本身的错误率和纠正率

特点：

- 建议按周或双周产出
- 是平台持续优化的依据

## 四、保留时长建议

### 原始对话

- 每次实际送模型：最近 `8-12` 轮
- 服务端存档：最近 `50-100` 轮
- 超出部分：压缩进项目记忆或用户画像，不再直接喂模型

### 项目级记忆

- 项目生命周期内长期保留
- 项目归档后转只读

### 用户级沟通画像

- 长期保留
- 每 `2-4` 周重算一次
- 当新行为和旧画像冲突时，优先参考最近一段时间的高置信度信号

### 周期总结

- 建议按 `7天` 或 `14天` 生成
- 保留最近 `8-12` 期即可

---

## 五、上下文压缩策略

### 策略 1：轮次压缩

每累计 `6-8` 轮旧对话时，压成四类结构：

- `事实`
- `偏好`
- `否定项`
- `未解决问题`

### 策略 2：事件压缩

在以下关键节点触发结构化总结：

- 用户导出
- 用户重生成
- 用户明确说“不要这样”
- 用户明确说“这个方向对”
- 用户大幅手动修改时间轴

### 策略 3：升级规则

只有满足以下条件的内容，才进入长期记忆：

- 用户明确表达过
- 或连续 `2-3` 次行为一致
- 或对最终导出结果产生了明显影响

### 策略 4：冲突解决

记忆冲突时按优先级处理：

1. 最新明确指令
2. 最近高频重复行为
3. 现有结构化记忆
4. 更早期的摘要

### 策略 5：置信度标注

每条长期记忆都应带：

- `source`：来自用户显式表达 / 行为推断 / 周期总结
- `confidence`：0-1
- `lastConfirmedAt`
- `evidenceCount`

低置信度记忆不应强注入模型，只能作为弱提示。

---

## 六、每次调用 Agent 时的上下文组装

推荐顺序：

1. 当前任务目标
2. 当前创意 brief
3. 项目级记忆摘要
4. 用户级沟通方式画像摘要
5. 最近 `8-12` 轮原始对话
6. 当前时间轴 / 素材 / 已选中内容摘要

关键原则：

- `项目记忆` 决定内容方向
- `用户画像` 决定 Agent 怎么表达、怎么确认、自动化到什么程度
- `原始对话` 只负责短期连续性

---

## 七、平台要记录的真实体验信号

为了后面做“基于真实用户体验的总结”，平台需要先把以下信号记录完整：

### 用户表达信号

- 用户输入的原始要求
- 用户对 Agent 的纠正
- 用户明确喜欢 / 不喜欢的表达

### Agent 输出信号

- Agent 给出的方案
- Agent 给出的 creative strategy
- Agent 给出的解释和确认方式

### 用户行为信号

- 是否采纳 Agent 方案
- 是否继续手动修改
- 修改幅度大不大
- 是否重生成
- 是否导出
- 是否放弃当前结果

### 结果信号

- 导出前重做次数
- 每条视频的返工次数
- 首次得到可用结果的时间
- 后续是否进入发布或分发流程

---

## 八、后续平台持续优化闭环

### 阶段 1：先做“记忆准确”

目标：

- 让系统先记对，不急着记太多

重点优化：

- 记忆生成规则
- 记忆冲突解决
- 负向偏好优先级
- 用户可编辑记忆

### 阶段 2：再做“记忆可解释”

目标：

- 让用户知道系统为什么这样回应、这样剪

重点优化：

- “为什么这么做”解释卡
- “系统记住了什么”面板
- “忘掉这个偏好”入口

### 阶段 3：再做“自动学习”

目标：

- 平台不只记忆，还能自己总结怎么变好

重点优化：

- 周报自动总结
- 画像变化趋势
- 不同用户群的协作方式聚类
- 哪种记忆最能提升采纳率

### 阶段 4：做“平台级优化建议”

目标：

- 让平台每隔一段时间主动告诉团队该优化哪里

输出形式：

- 周报 / 双周报
- 高频问题榜单
- 高价值功能建议
- 误判最多的场景
- 最值得自动化的下一步

---

## 九、推荐先做的产品能力

### P0

- 同项目历史持久化
- 项目级结构化记忆
- 用户级沟通画像
- 基础压缩与置信度机制

### P1

- 记忆面板：查看 / 编辑 / 删除
- “记住这个偏好”与“不要再这样做”显式反馈
- 导出节点自动总结

### P2

- 周期体验总结
- 用户群画像聚类
- 平台优化建议自动生成

---

## 十、北极星判断标准

这套系统是否有效，不看“存了多少历史”，而看下面这些指标：

- Agent 建议采纳率是否提升
- 用户返工次数是否下降
- 首次得到可用结果的时间是否缩短
- 用户纠正同类问题的频次是否下降
- 用户是否越来越少重复交代同一类偏好

---

## 结论

这套记忆系统的正确方向不是“做一个超长上下文聊天记录器”，而是：

- 用 `项目级记忆` 记住内容和剪辑偏好
- 用 `用户级记忆` 记住沟通方式画像
- 用 `周期总结` 驱动平台持续优化

一句话概括：

> 项目记忆管“做什么”，用户画像管“怎么合作”，周期总结管“平台下一步怎么变好”。
